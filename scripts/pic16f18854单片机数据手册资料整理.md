# pic16f18854单片机数据手册资料整理

# 7\.0 中断功能概述

中断功能允许特定事件中断正常程序流程，固件用于确定中断源并做出相应处理。部分中断可配置为将 MCU 从睡眠模式唤醒 。

**涵盖内容**

本章包含中断的以下信息：

- 操作（Operation）：介绍中断的工作原理。

- 中断延迟（Interrupt Latency）：涉及从中断事件发生到开始执行中断服务程序的时间间隔相关内容。

- 睡眠期间的中断（Interrupts During Sleep）：说明在 MCU 睡眠模式下中断的工作情况 。

- INT 引脚（INT Pin）：关于特定中断引脚的功能等。

- 自动上下文保存（Automatic Context Saving）：指 MCU 在响应中断时自动保存当前程序上下文（如寄存器值等），以便中断处理结束后能恢复继续执行原程序 。

**中断逻辑框图解读**

图 7 \- 1 展示了中断逻辑：

- 外设中断（Peripheral Interrupts）：众多外设可产生中断，如定时器 TMR1（由 TMR1IF 标志位和 TMR1IE 使能位控制 ）。每个外设中断都需使能对应标志位（PIRn）和使能位（PIEn），相关信号经逻辑与后进入或门 。

- 其他中断信号：

    - INT 中断：由 INTF 标志位和 INTE 使能位控制 。

    - 端口变化中断（IOC）：由 IOCIF 标志位和 IOCIE 使能位控制 。

- 使能控制：这些中断信号经或门后，还需使能外设中断使能位 PEIE 和全局中断使能位 GIE，最终才能向 CPU 发出中断请求。若 MCU 处于睡眠模式，符合条件的中断可将其唤醒 。

## 7\.1 中断操作（Operation）

- 中断使能：设备复位时中断默认禁用，需通过设置以下位来启用：

    - INTCON 寄存器的 GIE 位：全局中断使能位，置 1 开启全局中断允许。

    - 特定中断的中断使能位：如定时器中断、外部中断等各自对应的使能位 。

    - INTCON 寄存器的 PEIE 位（若中断使能位在 PIEx 寄存器中） ：外设中断使能位，开启后允许外设中断。

- PIR 寄存器：PIR1、PIR2、PIR3 和 PIR4 寄存器通过中断标志位记录单个中断，即便 GIE、PEIE 及个别中断使能位状态如何，中断标志位都会置 1 。

- 中断事件发生时的操作：当 GIE 位为 1 且中断事件发生时：

    - 清除当前预取指令：清除当前已预取但未执行的指令。

    - 清零 GIE 位：关闭全局中断，防止处理中断时新中断干扰。

    - 压栈操作：将当前程序计数器（PC）压入堆栈 。

    - 保存关键寄存器：自动将关键寄存器保存到影子寄存器（见 “7\.5 自动上下文保存”） 。

    - 加载中断向量：将 PC 加载为中断向量地址 0004h，程序跳转至中断服务程序（ISR） 。

    - 确定中断源并处理：固件通过轮询中断标志位确定中断源，在退出 ISR 前必须清除中断标志位，避免重复中断。

    - 退出中断服务程序：RETFIE 指令从堆栈弹出先前地址，从影子寄存器恢复保存的上下文，设置 GIE 位，退出 ISR 。

- 注意事项：

    - 单个中断标志位会置 1，与其他使能位状态无关。

    - GIE 位清零时忽略所有中断，GIE 位再次置 1 后，发生的中断会得到处理 。

## 7\.2 中断延迟（Interrupt Latency）

- 定义：中断延迟指从中断事件发生到中断向量处代码开始执行的时间 。

- 同步中断延迟：对于同步中断，延迟为 3 或 4 个指令周期 。

- 异步中断延迟：异步中断的延迟为 3 到 5 个指令周期，具体取决于中断发生时间，可参考图 7 \- 2 和图 7 \- 3 了解详情 。

**图 7 \- 2：中断延迟（Interrupt Latency）**

- 横轴与时间单位：横轴以指令周期为单位，将一个指令周期细分为 Q1 \- Q4 四个阶段，展示不同类型指令执行时中断发生的时间点及延迟情况。

- 中断采样：中断信号在 Q1 阶段被采样，一旦检测到中断，会依据当前执行指令的周期数产生不同延迟。

- 不同指令周期下的延迟：

    - 单周期指令：执行单周期指令时检测到中断，清除当前指令，插入两个空操作（NOP）指令，随后执行中断向量地址（0004h）处指令，总延迟为 3 个指令周期。

    - 双周期指令：执行双周期指令时检测到中断，同样清除当前指令，插入两个 NOP 指令，再执行中断向量处指令，延迟也是 3 个指令周期。

    - 三周期指令：执行三周期指令时，若在特定阶段检测到中断，可能需插入 3 个或 4 个 NOP 指令，延迟为 4 个指令周期 。

**图 7 \- 3：INT 引脚中断时序（INT Pin Interrupt Timing）**

- 信号标识：图中展示了 OSC1（时钟信号）、CLKOUT（时钟输出，并非在所有振荡模式下都可用 ）、INT 引脚信号、INTF（中断标志位）、GIE（全局中断使能位）、PC（程序计数器）等信号。

- 中断标志采样：INTF 标志位在每个 Q1 阶段被采样（注释 1 ）。

- 中断延迟：异步中断延迟为 3 \- 5 个指令周期（TCY），同步中断延迟为 3 \- 4 个指令周期（注释 2 ）。无论当前执行的是单周期还是双周期指令，延迟情况类似 。

- 其他说明：

    - CLKOUT 并非在所有振荡模式下都存在（注释 3 ）。

    - 关于 INT 脉冲最小宽度，参考 “37\.0 电气规格” 部分（注释 4 ）。

    - INTF 标志位可在 Q4 \- Q1 周期内的任意时刻被使能置位（注释 5 ）

## 7\.3 睡眠期间的中断（Interrupts During Sleep）

- 唤醒条件：部分中断可用于将单片机从睡眠模式唤醒。要实现唤醒，外设需能在无系统时钟时运行，且在进入睡眠模式前，必须设置好中断源对应的中断使能位 。

- 唤醒后的处理：从睡眠模式唤醒后，若全局中断使能位 GIE 也已设置，处理器将跳转至中断向量处执行中断服务程序；若 GIE 未设置，处理器将在睡眠指令之后继续执行后续指令。在跳转至中断服务程序之前，睡眠指令后的第一条指令总会先执行。更多详情可参考 “8\.0 节能操作模式” 章节 。

## 7\.4 INT 引脚（INT Pin）

- 中断触发方式：INT 引脚可用于产生异步边沿触发中断。通过设置 PIE0 寄存器的 INTE 位来使能该中断 。由 INTCON 寄存器的 INTEDG 位决定中断触发的边沿：INTEDG 位为 1 时，上升沿触发中断；INTEDG 位为 0 时，下降沿触发中断 。

- 中断标志及响应：当 INT 引脚上出现有效边沿时，PIR0 寄存器的 INTF 位将被置位。若此时 GIE 和 INTE 位也已设置，处理器将把程序执行重定向到中断向量处 。

## 7\.5 自动上下文保存（Automatic Context Saving）

- 进入中断时：进入中断时，返回的程序计数器（PC）地址会被保存到堆栈中。此外，以下寄存器会自动保存到影子寄存器中：

    - W 寄存器 。

    - STATUS 寄存器（除了*TO*和*PD*位 ）。

    - BSR 寄存器 。

    - FSR 寄存器 。

    - PCLATH 寄存器 。

- 退出中断时：退出中断服务程序时，这些寄存器会自动恢复。在中断服务程序中对这些寄存器的任何修改都会丢失。若希望修改这些寄存器的值，应修改对应的影子寄存器，退出中断服务程序时其值会被恢复。影子寄存器位于第 31 存储区，可读写。根据用户应用需求，可能还需要手动保存其他寄存器 。

## 7\.6 寄存器描述

### 7\.6\.1 INTCON 寄存器功能概述

INTCON 寄存器用于控制单片机的中断功能，包含全局中断使能、外设中断使能以及中断触发边沿选择等功能位。

**寄存器位定义**

- bit 7 \- GIE（Global Interrupt Enable bit）：全局中断使能位。置 1 时，启用所有已激活的中断；置 0 时，禁用所有中断 。

- bit 6 \- PEIE（Peripheral Interrupt Enable bit）：外设中断使能位。置 1 时，启用所有已激活的外设中断；置 0 时，禁用所有外设中断 。

- bit 5 \- 1：未使用位，读取值为 0 。

- bit 0 \- INTEDG（Interrupt Edge Select bit）：中断边沿选择位。置 1 时，INT 引脚上升沿触发中断；置 0 时，INT 引脚下降沿触发中断 。

**注意事项**

中断标志位在中断条件满足时会被置位，与对应使能位或全局使能位 GIE 的状态无关。在启用中断前，用户软件需确保清除相应的中断标志位，以避免误触发中断。

### 7\.6\.2 PIE1寄存器功能概述

PIE1 寄存器用于使能特定的外设中断，只有当该寄存器中的对应位被置 1，且 INTCON 寄存器中的 PEIE 位也为 1 时，相应的外设中断才能被启用。

**寄存器位定义**

- bit 7 \- OSFIE（Oscillator Fail Interrupt Enable bit）：振荡器故障中断使能位。置 1 时，启用振荡器故障中断；置 0 时，禁用振荡器故障中断 。

- bit 6 \- CSWIE（Clock Switch Complete Interrupt Enable bit）：时钟切换完成中断使能位。置 1 时，启用时钟切换模块中断；置 0 时，禁用时钟切换模块中断 。

- bit 5 \- 2：未使用位，读取值为 0 。

- bit 1 \- ADTIE（Analog \- to \- Digital Converter \(ADC\) Threshold Compare Interrupt Enable bit）：模数转换器（ADC）阈值比较中断使能位。置 1 时，启用 ADC 阈值比较中断；置 0 时，禁用 ADC 阈值比较中断 。

- bit 0 \- ADIE（Analog \- to \- Digital Converter \(ADC\) Interrupt Enable bit）：模数转换器（ADC）中断使能位。置 1 时，启用 ADC 中断；置 0 时，禁用 ADC 中断 。

**注意事项**

要启用 PIE1 \- PIE8 寄存器控制的任何外设中断，必须将 INTCON 寄存器中的 PEIE 位设置为 1 。

### 7\.6\.3 PIE2寄存器功能概述

PIE2 寄存器用于使能特定的外设中断，和 PIE1 等其他外设中断使能寄存器配合，共同管理外设中断的开启与关闭。同样，要启用 PIE2 中控制的外设中断，需同时满足 INTCON 寄存器中的 PEIE 位为 1 。

**寄存器位定义**

- bit 7：未使用位，读取值为 0 。

- bit 6 \- ZCDIE（Zero \- Cross Detection \(ZCD\) Interrupt Enable bit）：过零检测（ZCD）中断使能位。置 1 时，启用过零检测中断；置 0 时，禁用过零检测中断 。

- bit 5 \- 2：未使用位，读取值为 0 。

- bit 1 \- C2IE（Comparator C2 Interrupt Enable bit）：比较器 C2 中断使能位。置 1 时，启用比较器 C2 中断；置 0 时，禁用比较器 C2 中断 。

- bit 0 \- C1IE（Comparator C1 Interrupt Enable bit）：比较器 C1 中断使能位。置 1 时，启用比较器 C1 中断；置 0 时，禁用比较器 C1 中断 。

**注意事项**

要启用 PIE1 \- PIE8 寄存器控制的任何外设中断，必须将 INTCON 寄存器中的 PEIE 位设置为 1 。

### 7\.6\.4 PIE3寄存器功能概述

PIE3 寄存器用于使能特定的外设中断，主要涉及串口通信（USART）和同步串行端口（MSSP）相关的中断。要启用 PIE3 中控制的外设中断，需同时满足 INTCON 寄存器中的 PEIE 位为 1 。

**寄存器位定义**

- bit 7 \- 6：未使用位，读取值为 0 。

- bit 5 \- RCIE（USART Receive Interrupt Enable bit）：通用同步异步收发器（USART）接收中断使能位。置 1 时，启用 USART 接收中断；置 0 时，禁用 USART 接收中断 。

- bit 4 \- TXIE（USART Transmit Interrupt Enable bit）：USART 发送中断使能位。置 1 时，启用 USART 发送中断；置 0 时，禁用 USART 发送中断 。

- bit 3 \- BCL2IE（MSSP2 Bus Collision Interrupt Enable bit）：MSSP2 总线冲突中断使能位。置 1 时，启用 MSSP2 总线冲突中断；置 0 时，禁用 MSSP2 总线冲突中断 。

- bit 2 \- SSP2IE（Synchronous Serial Port \(MSSP2\) Interrupt Enable bit）：MSSP2 同步串行端口中断使能位。置 1 时，启用 MSSP2 中断；置 0 时，禁用 MSSP2 中断 。

- bit 1 \- BCL1IE（MSSP1 Bus Collision Interrupt Enable bit）：MSSP1 总线冲突中断使能位。置 1 时，启用 MSSP1 总线冲突中断；置 0 时，禁用 MSSP1 总线冲突中断 。

- bit 0 \- SSP1IE（Synchronous Serial Port \(MSSP1\) Interrupt Enable bit）：MSSP1 同步串行端口中断使能位。置 1 时，启用 MSSP1 中断；置 0 时，禁用 MSSP1 中断 。

**注意事项**

要启用 PIE1 \- PIE8 寄存器控制的任何外设中断，必须将 INTCON 寄存器中的 PEIE 位设置为 1 。

### 7\.6\.5 PIE4寄存器功能概述

PIE4 寄存器用于使能特定定时器相关的外设中断。通过设置该寄存器中的相应位，可以开启或关闭对应定时器的中断功能。同样，要启用 PIE4 中控制的外设中断，需同时满足 INTCON 寄存器中的 PEIE 位为 1 。

**寄存器位定义**

- bit 7 \- 6：未使用位，读取值为 0 。

- bit 5 \- TMR6IE（TMR6 to PR6 Match Interrupt Enable bit）：定时器 6 与周期寄存器 6 匹配中断使能位。置 1 时，启用定时器 6 与 PR6 匹配中断；置 0 时，禁用该中断 。

- bit 4 \- TMR5IE（Timer5 Overflow Interrupt Enable bit）：定时器 5 溢出中断使能位。置 1 时，启用定时器 5 溢出中断；置 0 时，禁用该中断 。

- bit 3 \- TMR4IE（TMR4 to PR4 Match Interrupt Enable bit）：定时器 4 与周期寄存器 4 匹配中断使能位。置 1 时，启用定时器 4 与 PR4 匹配中断；置 0 时，禁用该中断 。

- bit 2 \- TMR3IE（TMR3 Overflow Interrupt Enable bit）：定时器 3 溢出中断使能位。置 1 时，启用定时器 3 溢出中断；置 0 时，禁用该中断 。

- bit 1 \- TMR2IE（TMR2 to PR2 Match Interrupt Enable bit）：定时器 2 与周期寄存器 2 匹配中断使能位。置 1 时，启用定时器 2 与 PR2 匹配中断；置 0 时，禁用该中断 。

- bit 0 \- TMR1IE（Timer1 Overflow Interrupt Enable bit）：定时器 1 溢出中断使能位。置 1 时，启用定时器 1 溢出中断；置 0 时，禁用该中断 。

**注意事项**

要启用 PIE1 \- PIE8 寄存器控制的任何外设中断，必须将 INTCON 寄存器中的 PEIE 位设置为 1 。

### 7\.6\.6 PIE5寄存器功能概述

PIE5 寄存器用于使能特定的外设中断，主要涉及 CLC（可配置逻辑单元）相关中断以及定时器门控采集相关中断。和其他外设中断使能寄存器一样，要启用 PIE5 中控制的外设中断，需同时满足 INTCON 寄存器中的 PEIE 位为 1 。

**寄存器位定义**

- bit 7 \- CLC4IE（CLC4 Interrupt Enable bit）：可配置逻辑单元 4 中断使能位。置 1 时，启用 CLC4 中断；置 0 时，禁用 CLC4 中断 。

- bit 6 \- CLC3IE（CLC3 Interrupt Enable bit）：可配置逻辑单元 3 中断使能位。置 1 时，启用 CLC3 中断；置 0 时，禁用 CLC3 中断 。

- bit 5 \- CLC2IE（CLC2 Interrupt Enable bit）：可配置逻辑单元 2 中断使能位。置 1 时，启用 CLC2 中断；置 0 时，禁用 CLC2 中断 。

- bit 4 \- CLC1IE（CLC1 Interrupt Enable bit）：可配置逻辑单元 1 中断使能位。置 1 时，启用 CLC1 中断；置 0 时，禁用 CLC1 中断 。

- bit 3：未使用位，读取值为 0 。

- bit 2 \- TMR5GIE（Timer5 Gate Interrupt Enable bit）：定时器 5 门控采集中断使能位。置 1 时，启用定时器 5 门控采集中断；置 0 时，禁用该中断 。

- bit 1 \- TMR3GIE（Timer3 Gate Interrupt Enable bit）：定时器 3 门控采集中断使能位。置 1 时，启用定时器 3 门控采集中断；置 0 时，禁用该中断 。

- bit 0 \- TMR1GIE（Timer1 Gate Interrupt Enable bit）：定时器 1 门控采集中断使能位。置 1 时，启用定时器 1 门控采集中断；置 0 时，禁用该中断 。

**注意事项**

要启用 PIE1 \- PIE8 寄存器控制的任何外设中断，必须将 INTCON 寄存器中的 PEIE 位设置为 1 。

### 7\.6\.7 PIE6寄存器功能概述

PIE6 寄存器用于使能与 CCP（捕捉 / 比较 / PWM）模块相关的外设中断。通过设置该寄存器中的相应位，可以开启或关闭对应 CCP 模块的中断功能。同样，要启用 PIE6 中控制的外设中断，需同时满足 INTCON 寄存器中的 PEIE 位为 1 。

**寄存器位定义**

- bit 7 \- 5：未使用位，读取值为 0 。

- bit 4 \- CCP5IE（CCP5 Interrupt Enable bit）：CCP 模块 5 中断使能位。置 1 时，启用 CCP5 中断；置 0 时，禁用 CCP5 中断 。

- bit 3 \- CCP4IE（CCP4 Interrupt Enable bit）：CCP 模块 4 中断使能位。置 1 时，启用 CCP4 中断；置 0 时，禁用 CCP4 中断 。

- bit 2 \- CCP3IE（CCP3 Interrupt Enable bit）：CCP 模块 3 中断使能位。置 1 时，启用 CCP3 中断；置 0 时，禁用 CCP3 中断 。

- bit 1 \- CCP2IE（CCP2 Interrupt Enable bit）：CCP 模块 2 中断使能位。置 1 时，启用 CCP2 中断；置 0 时，禁用 CCP2 中断 。

- bit 0 \- CCP1IE（CCP1 Interrupt Enable bit）：CCP 模块 1 中断使能位。置 1 时，启用 CCP1 中断；置 0 时，禁用 CCP1 中断 。

**注意事项**

要启用 PIE1 \- PIE8 寄存器控制的任何外设中断，必须将 INTCON 寄存器中的 PEIE 位设置为 1 。

### 7\.6\.8 PIE7寄存器功能概述

PIE7 寄存器用于使能特定的外设中断，涉及扫描器、CRC 计算、非易失性存储器（NVM）、数控振荡器（NCO）以及互补波形发生器（CWG）等相关中断。要启用 PIE7 中控制的外设中断，需同时满足 INTCON 寄存器中的 PEIE 位为 1 。

**寄存器位定义**

- bit 7 \- SCANIE（Scanner Interrupt Enable bit）：扫描器中断使能位。置 1 时，启用扫描器中断；置 0 时，禁用扫描器中断 。

- bit 6 \- CRCIE（CRC Interrupt Enable bit）：CRC（循环冗余校验）中断使能位。置 1 时，启用 CRC 中断；置 0 时，禁用 CRC 中断 。

- bit 5 \- NVMIE（NVM Interrupt Enable bit）：非易失性存储器任务完成中断使能位。置 1 时，启用 NVM 任务完成中断；置 0 时，禁用该中断 。

- bit 4 \- NCO1IE（NCO Interrupt Enable bit）：数控振荡器溢出中断使能位。置 1 时，启用 NCO 溢出中断；置 0 时，禁用该中断 。

- bit 3：未使用位，读取值为 0 。

- bit 6 \- CRCIF（CRC Interrupt Flag bit）：CRC 中断标志位。置 1 时，表示操作已完成（BUSY 从 1 变为 0 ）；置 0 时，表示无操作挂起或操作仍在进行中 。

- bit 5 \- NVMIF（Non \- Volatile Memory \(NVM\) Interrupt Flag bit）：非易失性存储器（NVM）中断标志位。置 1 时，表示请求的 NVM 操作已完成；置 0 时，表示 NVM 中断未触发 。

- bit 4 \- NCO1IF（Numerically Controlled Oscillator \(NCO\) Interrupt Flag bit）：数控振荡器（NCO）中断标志位。置 1 时，表示 NCO 已溢出；置 0 时，表示未发生相关中断事件 。

- bit 3：未使用位，读取值为 0 。

- bit 2 \- CWG3IF（CWG3 Interrupt Flag bit）：互补波形发生器 3 中断标志位。置 1 时，表示 CWG3 已进入关闭状态；置 0 时，表示 CWG3 正常运行或中断已清除 。

- bit 1 \- CWG2IF（CWG2 Interrupt Flag bit）：互补波形发生器 2 中断标志位。置 1 时，表示 CWG2 已进入关闭状态；置 0 时，表示 CWG2 正常运行或中断已清除 。

- bit 0 \- CWG1IF（CWG1 Interrupt Flag bit）：互补波形发生器 1 中断标志位。置 1 时，表示 CWG1 已进入关闭状态；置 0 时，表示 CWG1 正常运行或中断已清除 。

**注意事项**

中断标志位在中断条件满足时会被置位，与对应使能位或全局使能位 GIE 的状态无关。在启用中断前，用户软件需确保清除相应的中断标志位，以避免误触发中断。

### 7\.6\.19 PIR8寄存器功能概述

PIR8 寄存器用于记录与 SMT（可能是特定的定时或测量模块 ）相关的外设中断请求状态。当中断条件满足时，对应的标志位会被硬件置 1，与中断使能位或全局中断使能位 GIE 的状态无关。在处理或启用相关中断前，软件需确保清除相应的中断标志位，以避免误触发中断。

**寄存器位定义**

- bit 7 \- 6：未使用位，读取值为 0 。

- bit 5 \- SMT2PWAIF（SMT2 Pulse Width Acquisition Interrupt Flag bit）：SMT2 脉宽采集中断标志位。置 1 时，表示中断挂起；置 0 时，表示无中断挂起 。

- bit 4 \- SMT2PRAIF（SMT2 Period Acquisition Interrupt Flag bit）：SMT2 周期采集中断标志位。置 1 时，表示中断挂起；置 0 时，表示无中断挂起 。

- bit 3 \- SMT2IF（SMT2 Overflow Interrupt Flag bit）：SMT2 溢出中断标志位。置 1 时，表示发生 SMT2 溢出事件，需在软件中清除；置 0 时，表示未检测到溢出事件 。

- bit 2 \- SMT1PWAIF（SMT1 Pulse Width Acquisition Interrupt Flag bit）：SMT1 脉宽采集中断标志位。置 1 时，表示中断挂起；置 0 时，表示无中断挂起 。

- bit 1 \- SMT1PRAIF（SMT1 Period Acquisition Interrupt Flag bit）：SMT1 周期采集中断标志位。置 1 时，表示中断挂起；置 0 时，表示无中断挂起 。

- bit 0 \- SMT1IF（SMT1 Overflow Interrupt Flag bit）：SMT1 溢出中断标志位。置 1 时，表示发生 SMT1 溢出事件，需在软件中清除；置 0 时，表示未检测到溢出事件 。

**注意事项**

中断标志位在中断条件满足时会被置位，与对应使能位或全局使能位 GIE 的状态无关。在启用中断前，用户软件需确保清除相应的中断标志位，以避免误触发中断。

### 7\.6\.20 寄存器功能概述

- 中断控制寄存器（INTCON ）：用于全局和外设中断使能控制，以及 INT 引脚中断触发边沿选择。GIE 控制所有中断使能，PEIE 控制外设中断使能，INTEDG 选择 INT 引脚触发边沿。

- 外设中断使能寄存器（PIE0 \- PIE8 ）：分别用于使能不同外设相关的中断。如 PIE1 控制振荡器故障、时钟切换、ADC 相关等中断使能；PIE4 控制多个定时器相关中断使能等。

- 外设中断请求寄存器（PIR0 \- PIR8 ）：用于记录不同外设中断的请求状态，当中断条件满足时，对应标志位被硬件置 1，软件需在处理或启用中断前清除标志位，避免误触发。

**各寄存器位说明**

- INTCON：bit7（GIE）、bit6（PEIE）、bit0（INTEDG）有对应功能，其余未使用位读值为 0 。

- PIE0 \- PIE8：不同位对应不同外设中断使能控制，如 PIE3 的 bit5（RCIE）控制 USART 接收中断使能，bit4（TXIE）控制发送中断使能等 。

- PIR0 \- PIR8：各标志位记录对应外设中断请求状态，如 PIR4 的 bit5（TMR6IF）记录定时器 6 中断标志等 。

**应用要点**

在使用中断功能时，需先通过 INTCON 寄存器开启全局或外设中断使能，再利用 PIE 系列寄存器使能具体外设中断；同时，要关注 PIR 系列寄存器中标志位状态，及时清除标志位，以正确处理中断请求。

# 19\.0 PWM 模块功能概述

**模块功能**

- PWM 模块：PWMx 模块可生成频率和占空比可变的脉宽调制信号。PIC16 \(L\) F18854 除 CCP 模块外，还额外有 PWM6 和 PWM7 两个 PWM 模块，功能与 CCP 模块的 PWM 功能类似，只是无捕获或比较功能。

- 注：PWM6 和 PWM7 模块本质相同，文档中通用的 PWMx ，编程时 x 需替换为 6 或 7 ，如 PWMxCON 对应实际的 PWM6CON 和 PWM7CON ，PWMxEN 对应 PWM6EN 和 PWM7EN 。

**PWM 原理**

- 定义：PWM 是通过在完全导通和完全关断状态间快速切换，为负载供电的方案。PWM 信号为方波，高电平部分为 “导通” 状态（脉冲宽度） ，低电平部分为 “关断” 状态。

- 占空比：占空比指 “导通” 时间与 “关断” 时间的比例，以百分比表示 ，0% 表示完全关断，100% 表示完全导通。占空比越低，负载供电越少；占空比越高，负载供电越多 。

- 周期：PWM 周期指一个完整循环的持续时间，即导通时间与关断时间总和。

- 分辨率：PWM 分辨率指一个 PWM 周期内可呈现的最大步数，分辨率越高，对脉冲宽度及负载供电控制越精确。 图 19 \- 1 展示了 PWM 信号典型波形。

**功能概述**

标准 PWM 模式可在 PWMx 引脚生成脉宽调制信号，分辨率最高达 10 位。其周期、占空比和分辨率由 TMR2 寄存器、PR2 寄存器、PWMxCON 寄存器、PWMxDCH 寄存器、PWMxDCL 寄存器控制 。

**关键寄存器作用**

- TMR2 寄存器：定时器 2 寄存器，作为 PWM 时基计数，其计数与 PR2 寄存器配合确定 PWM 周期。

- PR2 寄存器：设定 TMR2 的周期值，当 TMR2 计数达到 PR2 设定值时产生溢出，触发 PWM 周期相关操作。

- PWMxCON 寄存器：PWM 控制寄存器，用于配置 PWM 相关功能，如使能 PWM 输出（PWMEN 位）、设置输出极性（PWMxPOL 位 ）等。

- PWMxDCH 和 PWMxDCL 寄存器：共同设置 PWM 的占空比，组成 10 位占空比数值。

**输出特性**

- 当 PWMxPOL = 0 时，输出默认状态为‘0’；当 PWMxPOL = 1 时，输出默认状态为‘1’ 。若 PWMEN = 0，输出将保持默认状态。

- 要使能 PWMx 引脚输出，需将对应的 TRIS 位清零（设为输出模式） 。

**简化框图说明**

- 占空比寄存器（PWMxDCH 和 PWMxDCL）：设定占空比数值，经 10 位锁存器（用户不可见）传递给比较器。

- TMR2 模块：包括 TMR2 寄存器、比较器及 PR2 寄存器。TMR2 计数与 PR2 比较，产生 T2\_match 信号，用于控制 PWM 周期；同时与占空比数值比较，决定输出状态切换。

- PWMxPOL：控制输出极性。

- PPS（外设引脚选择）和 TRIS Control：PPS 用于将 PWM 输出映射到具体引脚，TRIS 控制引脚的输入输出模式。

## 19\.2 寄存器功能概述

### **19\.2\.1 PWMxCON**  寄存器概述

- 名称与功能：PWMxCON 为 PWM 控制寄存器，用于控制 PWM 模块的使能、输出电平读取、输出极性选择等功能。

- 位定义规则：文档中对寄存器各位的读写属性等进行了标注，如 R = 可读位，W = 可写位，U = 未实现位（读为‘0’ ）等 。

**具体位功能**

- bit 7（PWMxEN）：PWM 模块使能位 。置 1 时启用 PWM 模块；置 0 时禁用 PWM 模块。

- bit 6：未实现位，读取值为‘0’。

- bit 5（PWMxOUT）：读取该位时，可获取 PWM 模块输出电平状态。

- bit 4（PWMxPOL）：PWM 输出极性选择位 。置 1 时 PWM 输出为低电平有效（active \- low）；置 0 时 PWM 输出为高电平有效（active \- high） 。

- bit 3 \- 0：均为未实现位，读取值为‘0’ 。

### **19\.2\.2 PWMxDCH **寄存器概述

- 名称与功能：PWMxDCH 是 PWM 占空比高 8 位寄存器 ，用于存储 PWM 占空比的高 8 位数值（PWMxDC\<9:2\> ） ，与 PWMxDCL 寄存器共同构成 10 位的 PWM 占空比设置值，决定 PWM 信号的占空比。

- 位定义规则：寄存器各位属性标注遵循特定规则，R = 可读位，W = 可写位 ，x 表示复位后该位值未知，u 表示复位后该位值不变 。

**具体位功能**

- bit 7 \- 0（PWMxDC\<9:2\>）：这些位是 PWM 占空比的高 8 位（最高有效位，MSBs） ，与 PWMxDCL 寄存器中的低 2 位（最低有效位，LSBs）一起组成完整的 10 位 PWM 占空比数值，通过对这些位进行读写操作，可设置 PWM 信号的占空比。

### **19\.2\.3 PWMxDCH **寄存器概述

- 名称与功能：PWMxDCL 是 PWM 占空比低 2 位寄存器 ，用于存储 PWM 占空比的低 2 位数值（PWMxDC\<1:0\> ） ，与 PWMxDCH 寄存器共同构成 10 位的 PWM 占空比设置值，决定 PWM 信号的占空比。

- 位定义规则：寄存器各位属性标注遵循特定规则，R = 可读位，W = 可写位 ，U 表示未实现位（读为‘0’ ） ，x 表示复位后该位值未知，u 表示复位后该位值不变 。

**具体位功能**

- bit 7 \- 6（PWMxDC\<1:0\>）：这些位是 PWM 占空比的低 2 位（最低有效位，LSBs） ，与 PWMxDCH 寄存器中的高 8 位（最高有效位，MSBs）一起组成完整的 10 位 PWM 占空比数值，通过对这些位进行读写操作，可设置 PWM 信号的占空比。

- bit 5 \- 0：均为未实现位，读取值为‘0’ 。

# 29\.0 Timer2、Timer4、Timer6模块功能概述

**模块基本信息**

- 功能概述：Timer2/4/6 模块为 8 位定时器，可作为自由运行周期计数器，也可在单触发（One\-Shot）和单稳态（Monostable）操作模式下，配合外部信号控制启动、运行、冻结和复位 。通过与比较器、CCP 模块等其他内部外设协同，可实现脉冲密度调制等复杂波形控制。

- 模块一致性：芯片上有三个相同的 Timer2 模块，分别命名为 Timer2、Timer4、Timer6 ，对 Timer2 及 T2PR 的相关描述同样适用于 Timer4、Timer6 和 T4PR、T6PR 。

**模块特性**

- 寄存器：包含 8 位定时器寄存器和 8 位周期寄存器。

- 外部控制：支持可选的外部硬件定时器复位。

- 预分频与后分频：可编程预分频器（比例 1:1 至 1:128 ），可编程后分频器（比例 1:1 至 1:16 ）。

- 运行模式：可选同步 / 异步操作，有备用时钟源。

- 中断功能：周期匹配时可产生中断。

**操作模式**

- 自由运行周期（Free Running Period） ：定时器持续计数，达到周期寄存器设定值后自动重置并继续计数。

- 单触发（One\-shot） ：在外部信号触发下开始计数，计满后停止，需再次触发才能重新计数。

- 单稳态（Monostable） ：受外部信号触发，按设定时间输出稳定电平脉冲 。可参考 Figure 29 \- 1（Timer2 框图）和 Figure 29 \- 2（时钟源框图）了解更多细节。

### 29\.1 可配置逻辑单元（CLC）模块功能概述定时器2工作模式

- 自由运行周期模式（Free Running Period Mode） ：TMR2 与周期寄存器 T2PR 每个时钟周期比较，相等时 TMR2 在下一周期清零，后分频器计数递增。后分频器计数值等于 TMRxCON1 寄存器的 OUTPS\<4:0\> 值时，TMR2 后分频输出产生一个时钟周期宽脉冲，后分频计数清零 。

- 单触发模式（One \- Shot Mode） ：类似于自由运行周期模式，但 ON 位清零，TMR2 匹配 T2PR 后停止计数，直到 T2ON 位再次翻转才重启，此模式下非零的后分频器 OUTPS\<4:0\> 值无意义，定时器重启时后分频器复位 。

- 单稳态模式（Monostable Mode） ：与单触发模式类似，区别在于 ON 位不清零，定时器可由外部复位事件重启 。各模式下有多种启动、停止和复位选项，外部硬件信号可控制定时器运行或强制 TMR2 计数复位 。

**时钟源**

- 图 29 \- 2 展示 Timer2 时钟源框图，通过 TxCLKCON 和 TxINPPS 等配置，可选择不同时钟源（Timer Clock Sources ），具体见寄存器 29 \- 1 。

**寄存器特性**

- TMR2 寄存器可读可写，设备复位时双缓冲且初始化为0*xFF* 。T2PR 寄存器直接读写，实际周期缓冲器在特定事件（写 TMR2、T2CON、T2HLT 寄存器，TMR2 = T2PR 且预分频器满，外部复位事件 ）发生时更新 。预分频器和后分频器在写 TMR2、T2CON 寄存器，设备复位，外部复位事件时清零 。T2CON 写入时 TMR2 不清零 。

### **29\.2 Timer2 输出**

- 主要输出 TMR2\_postscaled，后分频器计数匹配 TMR2 时钟周期时产生脉冲，可作为 ADC 模块自动转换触发、CWG 自动关闭源、Memory Scanner 触发、Timer 1/3 门控输入、Timer 2/4/6 外部复位信号、SMT 窗口和信号输入 。此外，CCP 模块在 PWM 模式生成脉冲时也会用到 Timer2 相关信号 。

### **29\.3 Timer2 输出**

**外部复位源功能**

- Timer2 除时钟源外，还可接入外部复位源。Timer2、Timer4、Timer6 分别通过 T2RST、T4RST、T6RST 寄存器选择外部复位源 。该外部复位源可根据定时器工作模式，控制定时器的启动、停止和复位 。

**触发模式要求**

- 边沿触发模式：外部触发信号之间需间隔六个定时器时钟周期。

- 电平触发模式：触发电平持续时间至少为三个定时器时钟周期 。

**特殊模式下情况**

- 在调试冻结（Debug Freeze）模式下，外部触发信号将被忽略 。定时器工作模式由 TMRxHLT 寄存器的 MODE\<4:0\> 位控制 。

# 30\.0 捕获模式、比较模式、PWM 模式

**模块功能**

- 捕获模式（Capture mode）：用于测量事件持续时间，记录特定事件发生时刻。

- 比较模式（Compare mode）：当预设时间结束时，触发外部事件。

- PWM 模式（PWM mode）：可生成频率和占空比可变的脉宽调制信号 。

**模块配置**

- PIC16 \(L\) F18854 单片机具备 CCP1 \- CCP5 共 5 个 CCP 模块 ，表格 30 \- 1 展示了模块配置情况。

- 所有 CCP 模块的捕获和比较功能相同 。

**注意事项**

- 对于有多个 CCP 模块的设备，使用时需注意寄存器名称，模块缩写后的数字用于区分不同模块，如 CCP1CON 和 CCP2CON 分别控制不同 CCP 模块的相同操作方面 。

- 文档中对 CCP 模块通用描述，在各工作模式下同等适用于 CCPx 模块，寄存器名、模块信号、I/O 引脚和位名等可能用通用标识符 “x” ，必要时用数字区分特定模块 。

## 30\.1 捕获模式功能

- 所有 CCP 模块的捕获模式功能相同，利用 16 位 Timer1 资源。当捕获源发生指定事件时，16 位 CCPRxH:CCPRxL 寄存器对捕获并存储 TMR1H:TMR1L 寄存器对的 16 位值 。

- 事件类型由 CCPxCON 寄存器的 CCPxMODE\<3:0\> 位配置，包括每个下降沿、每个上升沿、每第 4 个上升沿、每第 16 个上升沿 。

**捕获源设置**

- CCPx 引脚需通过设置相关 TRIS 控制位配置为输入。若配置为输出，对端口的写操作可能导致捕获条件触发 。

- 捕获源通过配置 CCPxCAP 寄存器的 CCPxCTS\<2:0\> 位选择，可选项包括 CCPxPPS 输入、C1OUT\_sync、C2OUT\_sync、IOC\_interrupt、LC1\_out、LC2\_out、LC3\_out、LC4\_out 。图 30 \- 1 展示了捕获操作简化示意图。

- CCP 预分频器（30\.1\.4） ：CCPxCON 寄存器的 CCPxMODE\<3:0\> 位指定四个预分频器设置。CCP 模块关闭或复位时，预分频器计数器清零。切换预分频器设置不会清零计数器，可能产生虚假中断，为避免意外操作，更改预分频器前需清零 CCPxCON 寄存器 。

- 睡眠期间捕获（30\.1\.5） ：捕获模式依赖 Timer1 模块正常运行。Timer1 有两种驱动方式，由指令时钟（Fosc/4）或外部时钟源驱动。Fosc/4 时钟驱动时，睡眠期间 Timer1 不计数，设备唤醒后继续；外部时钟源驱动时，睡眠期间捕获模式可正常工作 。

- 比较模式不应使用系统时钟（Fosc）为 Timer1 计时，Timer1 需由指令时钟（Fosc/4）或外部时钟源驱动，以便 CCP 引脚识别触发事件 。

**自动转换触发（30\.2\.3）**

- 所有 CCPx 模式都会置位 CCP 中断标志（CCPxIF）。该标志置位且发生匹配时，若 CCP 模块被选为转换触发源，可触发自动转换 。更多信息参考 23\.2\.6 节 “自动转换触发” 。

- 在产生自动转换触发的时钟沿与产生 Timer1 复位的时钟沿之间，改变 CCPRxH 和 CCPRxL 寄存器对内容以移除匹配条件，会阻止 Timer1 复位发生 。

**睡眠期间比较（30\.2\.4）**

- 睡眠模式下 Fosc 关闭，除非定时器运行，否则比较模式无法正常工作。若中断使能，设备会在中断时唤醒 。

## 30\.3 PWM 原理

- 定义与波形：PWM 通过在完全导通和完全关断状态间快速切换为负载供电，信号类似方波，高电平为导通状态，低电平为关断状态。脉冲宽度（高电平部分）可变，其步长数量影响输出功率，步长越多，脉冲宽度越长，负载供电越多；反之越少。PWM 周期指一个完整循环持续时间，即导通与关断时间总和 。PWM 分辨率指单个 PWM 周期内最大步长数量，分辨率越高，对脉冲宽度及负载供电控制越精确 。占空比以百分比表示导通时间与关断时间比例，0% 为完全关断，100% 为完全导通，占空比高低对应负载供电少多 。图 30 \- 3 展示 PWM 信号典型波形 。

### 30\.4\.1 CCPxCON 寄存器（30 \- 1）

- 功能概述：用于配置和控制 CCP 模块的工作模式及相关功能。

- 位定义：

    - bit 7（EN） ：CCPx 模块使能位，1 启用模块，0 禁用。

    - bit 6 ：未实现位，读‘0’。

    - bit 5（OUT） ：CCPx 输出数据位，只读。

    - bit 4（FMT） ：PWM 脉冲宽度对齐位，1 左对齐，0 右对齐，捕获和比较模式未用。

    - bit 3 \- 0（MODE\<3:0\>） ：

        - PWM 模式：1111 。

        - 比较模式：多种编码对应不同输出动作，如 1011 输出脉冲 0 \- 1 \- 0 并清零 TMR1；1010 输出脉冲 0 \- 1 \- 0；1001 匹配时清零输出；1000 匹配时置位输出 。

        - 捕获模式：多种编码对应不同捕获事件，如 0111 捕获 CCPx 输入每个边沿；0110 捕获每 16 个上升沿；0101 捕获每个上升沿；0100 捕获每个下降沿 。

        - 0000 ：关闭 Capture/Compare/PWM 功能，复位 CCPx 模块 。

- 中断与 ADC 触发：所有模式都会置位 CCPxIF 中断标志位，若 CCPx 选为 ADC 触发源，还会触发 ADC 转换 。

### 30\.4\.2 CCPxCAP 寄存器（30 \- 2）

- 功能：捕获输入选择寄存器，用于选择 CCP 模块捕获触发输入源。

- 位定义：

    - bit 7 \- 3 ：未实现位，读‘0’。

    - bit 2 \- 0（CTS\<2:0\>） ：捕获触发输入选择位，不同编码对应不同触发源，如 000 对应 CCP1PPS \- CCP5PPS；001 对应 C1OUT；010 对应 C2OUT 等 。

### 30\.4\.3 CCPRxL 寄存器（30 \- 3）

- 功能：CCP 模块寄存器低字节，在不同模式下功能不同。

- 位定义：

    - CCPxMODE = 捕获模式 ：CCPRxL\<7:0\> 存储 TMR1L 的捕获值。

    - CCPxMODE = 比较模式 ：CCPRxL\<7:0\> 是与 TMR1L 比较的低字节。

    - CCPxMODE = PWM 模式 ：当 CCPxFMT = 0 时，CCPRxL\<7:0\> 是脉冲宽度低 8 位；当 CCPxFMT = 1 时，CCPRxL\<7:6 \> 是脉冲宽度低 2 位 。

### 30\.4\.4 CCPRxH 寄存器（30 \- 4）

- 功能：CCP 模块寄存器高字节，在不同模式下功能不同。

- 位定义：

    - CCPxMODE = 捕获模式 ：CCPRxH\<7:0\> 存储 TMR1H 的捕获值。

    - CCPxMODE = 比较模式 ：CCPRxH\<7:0\> 是与 TMR1H 比较的高字节。

    - CCPxMODE = PWM 模式 ：当 CCPxFMT = 0 时，CCPRxH\<7:2\> 未使用，CCPRxH\<1:0 \> 是脉冲宽度高 2 位；当 CCPxFMT = 1 时，CCPRxH\<7:0 \> 是脉冲宽度高 8 位 。

### 30\.4\.5 CCPTMRS0 寄存器（30 \- 5）

- 功能：CCP 定时器控制 0 寄存器，用于选择 CCP 模块所使用的定时器 。

- 位定义：

    - bit 7 \- 6（C4TSEL\<1:0\>） ：CCP4 定时器选择位，不同编码对应不同定时器，如 11 表示 CCP4 基于 TMR5（捕获 / 比较）或 TMR6（PWM） 。

    - bit 5 \- 4（C3TSEL\<1:0\>） ：CCP3 定时器选择位，编码规则同 C4TSEL 。

    - bit 3 \- 2（C2TSEL\<1:0\>） ：CCP2 定时器选择位，编码规则同 C4TSEL 。

    - bit 1 \- 0（C1TSEL\<1:0\>） ：CCP1 定时器选择位，编码规则同 C4TSEL 。

### 30\.4\.5 CCPTMRS1 寄存器功能

- CCPTMRS1 寄存器用于选择与 CCP 模块相关的定时器，以确定 CCP 模块工作时所使用的时基 。

**位定义**

- bit 7 \- 6 ：未实现位，读值为‘0’。

- bit 5 \- 4（P7TSEL\<1:0\>） ：PWM7 定时器选择位，不同编码对应不同定时器 ，如 11 表示 PWM7 基于 TMR6，10 表示基于 TMR4，01 表示基于 TMR2，00 为保留 。

- bit 3 \- 2（P6TSEL\<1:0\>） ：PWM6 定时器选择位，编码规则同 P7TSEL 。

- bit 1 \- 0（C5TSEL\<1:0\>） ：CCP5 定时器选择位，不同编码决定 CCP5 在捕获 / 比较或 PWM 模式下使用的定时器 ，如 11 表示基于 TMR5（捕获 / 比较）或 TMR6（PWM），10 表示基于 TMR3（捕获 / 比较）或 TMR4（PWM），01 表示基于 TMR1（捕获 / 比较）或 TMR2（PWM），00 为保留 。

# 22\.0 可配置逻辑单元（CLC）模块功能概述

CLC 模块可提供可编程逻辑功能，其运行速度不受软件执行速度限制。该逻辑单元可接收多达 32 个输入信号，通过可配置门电路，将 32 路输入精简为 4 路逻辑线路，进而驱动 8 种可选的单输出逻辑功能之一。

**输入源**

- I/O 引脚：可作为 CLC 模块的输入信号来源。

- 内部时钟：为 CLC 模块提供时钟相关的输入信号。

- 外设：其他外设模块的信号也能作为 CLC 模块的输入。

- 寄存器位：寄存器中的位信息可作为输入参与逻辑运算。

**输出方向**

CLC 模块的输出可在内部导向其他外设，也可导向输出引脚。

**可能配置**

- 组合逻辑：包括与（AND）、与非（NAND）、或（OR）、与或（AND \- OR）、与或非（AND \- OR \- INVERT）、或异或（OR \- XOR）、或同或（OR \- XNOR）等逻辑功能。

- 锁存器：有置位复位 S \- R 锁存器、带置位复位的时钟控制 D 锁存器、带置位复位的透明 D 锁存器、带复位的时钟控制 J \- K 锁存器等多种类型。

**模块实例**

PIC16 \(L\) F18854 单片机具备 CLC1、CLC2、CLC3、CLC4 四个可配置逻辑单元模块实例。文档中寄存器和位名称里的 “x” 为通用标识，实际编程时需根据具体模块（1、2、3 或 4 ）进行替换，如控制寄存器一般描述为 CLCxCON，实际设备寄存器则为 CLC1CON、CLC2CON、CLC3CON 和 CLC4CON 。

## 22\.1 CLCx 模块编程步骤

对 CLCx 模块编程需配置逻辑信号流中的四个阶段：

1. 数据选择：从 32 个可用输入信号中挑选进入下一阶段的输入。

2. 数据选通：对选择后的数据进行选通操作。

3. 逻辑功能选择：选择具体的逻辑功能对数据进行处理。

4. 输出极性设置：设置最终输出信号的极性。
各阶段可通过写入相应的 CLCx 特殊功能寄存器在运行时进行设置，便于在程序执行期间重新配置逻辑。

### **22\.1\.1 数据选择具体实现**

- 多路复用器：使用 4 个 32 输入多路复用器来选择输入。数据选择通过 CLCxSEL0 \- CLCxSEL3 寄存器（寄存器 22 \- 3 至寄存器 22 \- 6 ）进行。

### 22\.1\.2 数据选通（Data Gating）

- 功能原理：输入多路复用器的输出信号通过数据选通阶段，导向期望的逻辑功能输入。每个数据选通门可引导四个选定输入的任意组合。选通门不仅是信号导向，还能配置为将每个输入信号反向或不反向，输出也能反向或不反向。本质上是 1 \- 4 输入的与 / 与非 / 或 / 或非门。若所有输入反向且输出反向，门为或门；若输入和输出不反向，门为与门 。

- 配置方法：通过逻辑门选择寄存器配置，如 Gate 1 对应 CLCxGLS0（寄存器 22 \- 7 ）、Gate 2 对应 CLCxGLS1（寄存器 22 \- 8 ）等 。表格 22 \- 3 展示了数据选通逻辑，如 CLCxGLSx 值为 0x55 且 LCxGyPOL 为 1 时，门逻辑为与（AND）；CLCxGLSx 值为 0x55 且 LCxGyPOL 为 0 时，门逻辑为与非（NAND）等 。

- 注意事项：上电时数据选通未定义。不建议同时选择正负极性输入，否则门输出恒为 0，可能产生逻辑毛刺。推荐方法是将所有位设为 0，用门极性位设置期望电平 。

**逻辑功能选择（Logic Function）**

- 可选功能：有 8 种可用逻辑功能，包括与或（AND \- OR）、或异或（OR \- XOR）、与（AND）、S \- R 锁存器、带置位复位的 D 触发器、带复位的 D 触发器、带复位的 J \- K 触发器、带置位复位的透明锁存器 。

- 输入输出：每个逻辑功能有四个输入和一个输出，四个输入是前一阶段四个数据选通门的输出，输出送到反向阶段，再导向其他外设、输出引脚或返回 CLCx 本身 。

**输出极性设置（Output Polarity）**

可配置逻辑单元的最后阶段是输出极性设置。设置 CLCxPOL 寄存器的 LCxPOL 位可反转逻辑阶段的输出信号。在中断启用时改变极性，会因输出转换产生中断 。

### 22\.1\.2 CLCx 中断（CLCx Interrupts）

- 中断触发条件：当 CLCx 输出值变化且相应中断使能位已设置时，会产生中断。通过上升沿和下降沿检测器检测边沿变化，关联的 PIR5 寄存器中的 CLCxIF 位会在检测到使能边沿时置 1。

- 中断使能设置：要完全使能中断，需设置 PIE5 寄存器的 CLCxIE 位、CLCxCON 寄存器的 LCxINTP（上升沿）和 LCxINTN（下降沿）位，以及 INTCON 寄存器的 PEIE 和 GIE 位。CLCxIF 标志位需在软件中清除，若清除时检测到其他边沿，标志位仍会置 1 。

## **22\.3 输出镜像副本（Output Mirror Copies）**

所有 CLCxCON 寄存器的输出位都包含在 CLCxDATA 寄存器中，读取该寄存器可同时读取所有 CLC 的输出，避免测试或读取单个 CLCxCON 寄存器时引入的读取偏差 。

## **22\.4 复位影响（Effects of a Reset）**

复位会使 CLCxCON 寄存器清零，其他选择和选通值保持不变 。

## **22\.5 睡眠期间操作（Operation During Sleep）**

CLC 模块独立于系统时钟运行，睡眠期间若所选输入源保持活动，模块继续运行。选择 HFINTOSC 作为输入源时，睡眠期间其保持活动。若 HFINTOSC 同时作为系统时钟和 CLC 输入源，睡眠时 CPU 空闲，CLC 继续运行且 HFINTOSC 保持活动，这会影响睡眠模式电流 。

## **22\.6 CLCx 设置步骤（CLCx Setup Steps）**

1. 禁用 CLCx：清零 LCxEN 位。

2. 选择输入：通过 CLCxSEL0 \- CLCxSEL3 寄存器选择所需输入。

3. 清除相关位：清除相关 ANSEL 位，设置输入相关 TRIS 位，清零输出相关 TRIS 位。

4. 使能输入：通过 CLCxGLS0 \- CLCxGLS3 寄存器使能选定输入。

5. 设置门输出极性：用 LCxGyPOL 位选择门输出极性。

6. 选择逻辑功能：用 LCxMODE\<2:0\> 位选择逻辑功能。

7. 设置逻辑输出极性：用 LCxPOL 位选择逻辑输出极性。

8. 设置引脚相关：若驱动设备引脚，设置 PPS 控制寄存器并清零对应 TRIS 位。

9. 配置中断（若需要）：设置 LCxINTP（上升沿）、LCxINTN（下降沿）、CLCxIE、GIE 和 PEIE 位。

10. 启用 CLCx：设置 LCxEN 位 。

## 22\.7寄存器功能概述

### 22\.7\.1 CLCxCON 寄存器

用于控制 CLCx 模块的使能、输出数据读取、中断使能以及功能模式设置等。通过对该寄存器各个位的操作，可以实现对 CLCx 模块工作状态的精准配置。

**寄存器位定义**

- bit 7 \- LCxEN（Configurable Logic Cell Enable bit）：可配置逻辑单元使能位。置 1 时，启用可配置逻辑单元并混合输入信号；置 0 时，禁用可配置逻辑单元，输出为逻辑 0 。

- bit 6：未使用位，读取值为 0 。

- bit 5 \- LCxOUT（Configurable Logic Cell Data Output bit）：可配置逻辑单元数据输出位，只读。用于读取逻辑单元在经过 LCxPOL 极性设置后的输出数据，数据从 CLCxOUT 采样 。

- bit 4 \- LCxINTP（Configurable Logic Cell Positive Edge Going Interrupt Enable bit）：可配置逻辑单元上升沿中断使能位。置 1 时，当 CLCxOUT 出现上升沿，会设置 CLCxIF 中断标志位；置 0 时，不会设置该标志位 。

- bit 3 \- LCxINTN（Configurable Logic Cell Negative Edge Going Interrupt Enable bit）：可配置逻辑单元下降沿中断使能位。置 1 时，当 CLCxOUT 出现下降沿，会设置 CLCxIF 中断标志位；置 0 时，不会设置该标志位 。

- bit 2 \- 0 \- LCxMODE\<2:0\>（Configurable Logic Cell Functional Mode bits）：可配置逻辑单元功能模式位，不同编码对应不同功能：

    - 111：1 输入带置位复位的透明锁存器。

    - 110：带复位的 J \- K 触发器。

    - 101：2 输入带复位的 D 触发器。

    - 100：1 输入带置位复位的 D 触发器。

    - 011：S \- R 锁存器。

    - 010：4 输入与门。

    - 001：或 \- 异或功能。

    - 000：与 \- 或功能 。

### 22\.7\.2 CLCxPOL 寄存器功能概述

CLCxPOL 寄存器主要用于控制 CLCx 模块的输出信号极性以及数据选通门输出信号的极性。通过对该寄存器各个位的设置，可以灵活调整信号的极性，满足不同的逻辑需求。

**寄存器位定义**

- bit 7 \- LCxPOL（CLCxOUT Output Polarity Control bit）：CLCxOUT 输出极性控制位。置 1 时，逻辑单元的输出信号被反转；置 0 时，逻辑单元的输出信号不反转 。

- bit 6 \- 4：未使用位，读取值为 0 。

- bit 3 \- LCxG4POL（Gate 3 Output Polarity Control bit）：选通门 3 输出极性控制位。置 1 时，选通门 3 应用于逻辑单元的输出信号被反转；置 0 时，选通门 3 的输出信号不反转 。

- bit 2 \- LCxG3POL（Gate 2 Output Polarity Control bit）：选通门 2 输出极性控制位。置 1 时，选通门 2 应用于逻辑单元的输出信号被反转；置 0 时，选通门 2 的输出信号不反转 。

- bit 1 \- LCxG2POL（Gate 1 Output Polarity Control bit）：选通门 1 输出极性控制位。置 1 时，选通门 1 应用于逻辑单元的输出信号被反转；置 0 时，选通门 1 的输出信号不反转 。

- bit 0 \- LCxG1POL（Gate 0 Output Polarity Control bit）：选通门 0 输出极性控制位。置 1 时，选通门 0 应用于逻辑单元的输出信号被反转；置 0 时，选通门 0 的输出信号不反转 。

- bit 1 \- MLC2OUT（Mirror copy of LC2OUT bit）：LC2OUT 位的镜像副本，用于反映 CLC2 模块的输出状态 。
