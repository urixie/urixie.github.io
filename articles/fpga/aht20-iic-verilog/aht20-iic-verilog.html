Verilog 实现 AHT20 温湿度传感器读取

本文记录基于 Verilog 实现 AHT20 温湿度传感器读取的过程。整体设计包括 IIC 主机时序模块、AHT20 读取状态机、 CRC8/ITU 校验模块、温湿度 20bit 原始数据解析以及多次采样平均滤波。

AHT20 是标准 I2C 数字输出温湿度传感器，供电范围为 2.2V 到 5.5V。本文重点关注 FPGA 侧如何用 Verilog 直接实现 IIC 时序和传感器读取流程。

一、设计目标

  • 使用 FPGA 直接驱动 AHT20 温湿度传感器。
  • 系统时钟为 20MHz，IIC SCL 目标频率为 200kHz。
  • 实现 IIC 起始、停止、写字节、读字节、ACK、NACK 等基础操作。
  • 按照 AHT20 协议发送测量命令并读取 7 字节返回数据。
  • 对 Status、湿度数据、温度数据进行 CRC8/ITU 校验。
  • 解析 20bit 湿度原始值和 20bit 温度原始值。
  • 对最近 8 次采样结果做平均，降低瞬时抖动。

二、AHT20 通信流程

AHT20 上电后需要等待一段时间，再通过 IIC 发送测量命令。本文的状态机按照以下流程执行：

  1. 上电或复位后等待 5ms。
  2. 发送写地址 0x70。
  3. 发送测量命令 0xAC、0x33、0x00。
  4. 等待 80ms，等待传感器测量完成。
  5. 发送读地址 0x71。
  6. 连续读取 7 字节数据。
  7. 前 6 字节参与 CRC8 计算，第 7 字节为传感器返回的 CRC。
  8. CRC 正确后解析湿度和温度原始值。
  9. 等待 500ms 后重复下一次测量。
写测量命令：
0x70 0xAC 0x33 0x00

等待测量完成：
80ms

读传感器数据：
0x71

返回数据：
Status + SRH[19:0] + ST[19:0] + CRC

从逻辑分析仪解码结果可以看到，主机发出了写地址和 3 字节测量命令，数据段为： 

Address write：0x38
Data write：0xAC
Data write：0x33
Data write：0x00

从读取波形中可以看到，主机发送读地址后，AHT20 返回了 7 字节数据： 

Address read：0x38
Data read：18
Data read：2C
Data read：4F
Data read：35
Data read：C9
Data read：CE
Data read：01

其中第 1 字节是状态字，后续 5 字节包含 20bit 湿度原始值和 20bit 温度原始值，最后 1 字节为 CRC8 校验值。

三、IIC 主机模块设计

IIC 主机模块负责把上层命令转换成实际的 SDA / SCL 时序。命令位包括 WR、STA、RD、STO、ACK 和 NACK。 SDA 使用开漏方式实现：输出 0 时主动拉低，总线为 1 时释放为高阻态，由外部上拉电阻拉高。 

assign IO_i2c_sdat = (!i2c_sdat_o && i2c_sdat_oe) ? 1'b0 : 1'bz;

IIC 模块的主要状态包括：

  • IDLE：空闲状态。
  • GEN_STA：产生起始信号。
  • WR_DATA：写 8bit 数据。
  • RD_DATA：读 8bit 数据。
  • CHECK_ACK：检测从机 ACK。
  • GEN_ACK：主机产生 ACK 或 NACK。
  • GEN_STO：产生停止信号。
module IIC_MODULE(
    input I_clk,
    input I_rst_n,

    input [5:0]I_Cmd,
    input I_Go,
    output reg[7:0]O_Rx_DATA,
    input [7:0]I_Tx_DATA,
    output reg O_Trans_Done,
    output reg O_ack_o,
    output reg O_i2c_sclk,
    inout IO_i2c_sdat
);
    reg i2c_sdat_o;

    // 系统时钟采用20MHz
    parameter SYS_CLOCK = 20_000_000;
    // SCL总线时钟采用200kHz
    parameter SCL_CLOCK = 200_000;
    // 产生时钟SCL计数器最大值
    localparam SCL_CNT_M = SYS_CLOCK/SCL_CLOCK/4 - 1;

    reg i2c_sdat_oe;

    localparam WR   = 6'b000001;
    localparam STA  = 6'b000010;
    localparam RD   = 6'b000100;
    localparam STO  = 6'b001000;
    localparam ACK  = 6'b010000;
    localparam NACK = 6'b100000;

    reg [19:0]div_cnt;
    reg en_div_cnt;

    always@(posedge I_clk or negedge I_rst_n) begin
        if(!I_rst_n) begin
            div_cnt <= 20'd0;
        end
        else if(en_div_cnt)begin
            if(div_cnt < SCL_CNT_M) begin
                div_cnt <= div_cnt + 1'b1;
            end
            else begin
                div_cnt <= 0;
            end
        end
        else begin
            div_cnt <= 0;
        end
    end

    wire sclk_plus = (div_cnt == SCL_CNT_M) ? 1 : 0;

    assign IO_i2c_sdat = (!i2c_sdat_o && i2c_sdat_oe) ? 1'b0 : 1'bz;

    reg [7:0]state;

    localparam IDLE      = 8'b00000001;
    localparam GEN_STA   = 8'b00000010;
    localparam WR_DATA   = 8'b00000100;
    localparam RD_DATA   = 8'b00001000;
    localparam CHECK_ACK = 8'b00010000;
    localparam GEN_ACK   = 8'b00100000;
    localparam GEN_STO   = 8'b01000000;

    reg [4:0]cnt;

    always@(posedge I_clk or negedge I_rst_n)
    if(!I_rst_n)begin
        O_Rx_DATA <= 0;
        i2c_sdat_oe <= 1'd0;
        en_div_cnt <= 1'b0;
        i2c_sdat_o <= 1'd1;
        O_Trans_Done <= 1'b0;
        O_ack_o <= 0;
        state <= IDLE;
        cnt <= 0;
    end
    else begin
        case(state)
            IDLE: begin
                O_Trans_Done <= 1'b0;
                i2c_sdat_oe <= 1'd1;
                if(I_Go)begin
                    en_div_cnt <= 1'b1;
                    if(I_Cmd & STA) begin
                        state <= GEN_STA;
                    end
                    else if(I_Cmd & WR) begin
                        state <= WR_DATA;
                    end
                    else if(I_Cmd & RD) begin
                        state <= RD_DATA;
                    end
                    else begin
                        state <= IDLE;
                    end
                end
                else begin
                    en_div_cnt <= 1'b0;
                    state <= IDLE;
                end
            end

            GEN_STA: begin
                if(sclk_plus)begin
                    if(cnt == 3) begin
                        cnt <= 0;
                        if(I_Cmd & WR) begin
                            state <= WR_DATA;
                        end
                        else if(I_Cmd & RD) begin
                            state <= RD_DATA;
                        end
                    end
                    else begin
                        cnt <= cnt + 1'b1;
                    end

                    case(cnt)
                        0:begin i2c_sdat_o <= 1; i2c_sdat_oe <= 1'd1;end
                        1:begin O_i2c_sclk <= 1;end
                        2:begin i2c_sdat_o <= 0; O_i2c_sclk <= 1;end
                        3:begin O_i2c_sclk <= 0;end
                        default:begin i2c_sdat_o <= 1; O_i2c_sclk <= 1;end
                    endcase
                end
            end

            WR_DATA: begin
                if(sclk_plus)begin
                    if(cnt == 31) begin
                        cnt <= 0;
                        state <= CHECK_ACK;
                    end
                    else begin
                        cnt <= cnt + 1'b1;
                    end

                    case(cnt)
                        0,4,8,12,16,20,24,28:begin
                            i2c_sdat_o <= I_Tx_DATA[7-cnt[4:2]];
                            i2c_sdat_oe <= 1'd1;
                        end
                        1,5,9,13,17,21,25,29:begin
                            O_i2c_sclk <= 1;
                        end
                        2,6,10,14,18,22,26,30:begin
                            O_i2c_sclk <= 1;
                        end
                        3,7,11,15,19,23,27,31:begin
                            O_i2c_sclk <= 0;
                        end
                        default:begin
                            i2c_sdat_o <= 1;
                            O_i2c_sclk <= 1;
                        end
                    endcase
                end
            end

            RD_DATA: begin
                if(sclk_plus)begin
                    if(cnt == 31) begin
                        cnt <= 0;
                        state <= GEN_ACK;
                    end
                    else begin
                        cnt <= cnt + 1'b1;
                    end

                    case(cnt)
                        0,4,8,12,16,20,24,28:begin
                            i2c_sdat_oe <= 1'd0;
                            O_i2c_sclk <= 0;
                        end
                        1,5,9,13,17,21,25,29:begin
                            O_i2c_sclk <= 1;
                        end
                        2,6,10,14,18,22,26,30:begin
                            O_i2c_sclk <= 1;
                            O_Rx_DATA <= {O_Rx_DATA[6:0], IO_i2c_sdat};
                        end
                        3,7,11,15,19,23,27,31:begin
                            O_i2c_sclk <= 0;
                        end
                        default:begin
                            i2c_sdat_o <= 1;
                            O_i2c_sclk <= 1;
                        end
                    endcase
                end
            end

            CHECK_ACK: begin
                if(sclk_plus)begin
                    if(cnt == 3) begin
                        cnt <= 0;
                        if(I_Cmd & STO) begin
                            state <= GEN_STO;
                        end
                        else begin
                            state <= IDLE;
                            O_Trans_Done <= 1'b1;
                        end
                    end
                    else begin
                        cnt <= cnt + 1'b1;
                    end

                    case(cnt)
                        0:begin i2c_sdat_oe <= 1'd0; O_i2c_sclk <= 0;end
                        1:begin O_i2c_sclk <= 1;end
                        2:begin O_ack_o <= IO_i2c_sdat; O_i2c_sclk <= 1;end
                        3:begin O_i2c_sclk <= 0;end
                        default:begin i2c_sdat_o <= 1; O_i2c_sclk <= 1;end
                    endcase
                end
            end

            GEN_ACK: begin
                if(sclk_plus)begin
                    if(cnt == 3) begin
                        cnt <= 0;
                        if(I_Cmd & STO) begin
                            state <= GEN_STO;
                        end
                        else begin
                            state <= IDLE;
                            O_Trans_Done <= 1'b1;
                        end
                    end
                    else begin
                        cnt <= cnt + 1'b1;
                    end

                    case(cnt)
                        0:begin
                            i2c_sdat_oe <= 1'd1;
                            O_i2c_sclk <= 0;
                            if(I_Cmd & ACK) begin
                                i2c_sdat_o <= 1'b0;
                            end
                            else if(I_Cmd & NACK) begin
                                i2c_sdat_o <= 1'b1;
                            end
                        end
                        1:begin O_i2c_sclk <= 1;end
                        2:begin O_i2c_sclk <= 1;end
                        3:begin O_i2c_sclk <= 0;end
                        default:begin i2c_sdat_o <= 1; O_i2c_sclk <= 1;end
                    endcase
                end
            end

            GEN_STO: begin
                if(sclk_plus)begin
                    if(cnt == 3) begin
                        cnt <= 0;
                        O_Trans_Done <= 1'b1;
                        state <= IDLE;
                    end
                    else begin
                        cnt <= cnt + 1'b1;
                    end

                    case(cnt)
                        0:begin i2c_sdat_o <= 0; i2c_sdat_oe <= 1'd1;end
                        1:begin O_i2c_sclk <= 1;end
                        2:begin i2c_sdat_o <= 1; O_i2c_sclk <= 1;end
                        3:begin O_i2c_sclk <= 1;end
                        default:begin i2c_sdat_o <= 1; O_i2c_sclk <= 1;end
                    endcase
                end
            end

            default: state <= IDLE;
        endcase
    end

endmodule

四、CRC8/ITU 校验模块

AHT20 返回的最后 1 字节是 CRC。本文采用 CRC8/ITU 算法，初始值为 0xFF，多项式为 0x31。 FPGA 侧通过状态机逐 bit 计算 CRC，与传感器返回的 CRC 字节比较。 

module CRC8_ITU (
    input  wire [7:0] I_data_in,
    input  wire I_crc_en,
    input  wire I_rst,
    input  wire I_clk,
    output reg  [7:0] O_crc_out,
    output reg  [1:0] O_crc_busy
);

    reg [2:0] state;
    reg [4:0] cnt;

    always @(posedge I_clk) begin
        if (I_rst == 1'b0) begin
            O_crc_out <= 8'hFF;
            O_crc_busy <= 2'b00;
            cnt <= 4'd0;
            state <= 2'b0;
        end
        else begin
            case (state)
                2'b00: begin
                    if (I_crc_en == 1'b1) begin
                        cnt <= 4'd0;
                        state <= 2'b01;
                        O_crc_busy <= 2'b01;
                    end
                    else begin
                        state <= 2'b00;
                    end
                end

                2'b01: begin
                    O_crc_out <= O_crc_out ^ I_data_in;
                    state <= 2'b10;
                    O_crc_busy <= 2'b10;
                    cnt <= 4'd8;
                end

                2'b10: begin
                    if (cnt > 4'd0) begin
                        cnt <= cnt - 1;
                        O_crc_out <= (O_crc_out[7]) ? {O_crc_out[6:0], 1'b0} ^ 8'h31 : {O_crc_out[6:0], 1'b0};
                    end
                    else begin
                        O_crc_busy <= 2'b11;
                        state <= 2'b11;
                    end
                end

                2'b11: begin
                    if (I_crc_en == 1'b1) begin
                        state <= state;
                        O_crc_busy <= O_crc_busy;
                    end
                    else begin
                        O_crc_busy <= 2'b00;
                        state <= 2'b00;
                    end
                end
            endcase
        end
    end

endmodule

五、AHT20 顶层读取状态机

顶层模块负责按照 AHT20 的通信流程调度 IIC 模块。它会先发送测量命令，再等待 80ms， 然后读取 7 个字节并进行 CRC 校验。CRC 正确后提取湿度和温度原始数据。 

湿度原始值：
humidity_raw = {rx_buffer[1], rx_buffer[2], rx_buffer[3][7:4]}

温度原始值：
temperature_raw = {rx_buffer[3][3:0], rx_buffer[4], rx_buffer[5]}

为了降低瞬时抖动，代码中保存最近 8 次 humidity 和 temperature 原始值，并对其求和后右移 3 位， 得到平均后的 humidity_raw 和 temperature_raw。 

humidity_sum <= humidity_0 + humidity_1 + humidity_2 + humidity_3 +
                humidity_4 + humidity_5 + humidity_6 + humidity_7;

temperature_sum <= temperature_0 + temperature_1 + temperature_2 + temperature_3 +
                   temperature_4 + temperature_5 + temperature_6 + temperature_7;

humidity_raw <= humidity_sum[22:3];
temperature_raw <= temperature_sum[22:3];

六、关键状态机流程

顶层状态机的核心状态如下：

  • IDLE：进入初始化流程。
  • DELAY_5MS：上电等待。
  • SEND_INIT / WAIT_INIT：发送 0x70、0xAC、0x33、0x00。
  • DELAY_80MS：等待测量完成。
  • SEND_CMD / WAIT_CMD：发送 0x71 读命令。
  • RECEIVE / WAIT_RECV：读取 7 字节数据。
  • RX_DATA_CRC_PROC_START / RX_DATA_CRC_PROC：对前 6 字节做 CRC8。
  • PARSE_DATA：CRC 正确后解析数据。
  • REPEAT：等待 500ms 后重复测量。
localparam
    IDLE        = 4'd0,
    DELAY_5MS   = 4'd1,
    SEND_INIT   = 4'd2,
    WAIT_INIT   = 4'd3,
    DELAY_80MS  = 4'd4,
    SEND_CMD    = 4'd5,
    WAIT_CMD    = 4'd6,
    RECEIVE     = 4'd7,
    WAIT_RECV   = 4'd8,
    RX_DATA_CRC_PROC_START = 4'd9,
    RX_DATA_CRC_PROC = 4'd10,
    PARSE_DATA  = 4'd11,
    REPEAT      = 4'd12;

七、温湿度转换公式

AHT20 返回的是 20bit 原始值，需要在后级显示模块、MCU 或上位机中换算成实际温湿度。 常用换算公式如下： 

相对湿度：
RH = humidity_raw / 2^20 * 100%

温度：
T = temperature_raw / 2^20 * 200 - 50

FPGA 内部如果不需要直接显示小数，可以先保留 20bit 原始值或放大后的整数值，再交给显示层处理。

八、调试注意事项

AHT20 的 SDA 为双向开漏总线，FPGA 不能直接强推高电平。输出 1 时应释放为高阻态，由外部上拉电阻拉高。
  • SDA、SCL 需要外部上拉电阻。
  • AHT20 上电后建议等待 5ms 再通信。
  • 发送测量命令后应等待约 80ms 再读取。
  • 读取 7 字节时，前 6 字节参与 CRC8 校验。
  • 最后 1 字节读取后主机应发送 NACK 并产生 STOP。
  • 若读到状态字 Bit7 为 1，说明传感器仍在忙，可能读到上一次数据。
  • IIC 时序问题优先用逻辑分析仪观察 SDA、SCL。

九、源码说明

完整工程建议拆成三个文件：

  • iic.v：IIC 主机时序模块。
  • crc8_itu.v：CRC8/ITU 校验模块。
  • aht20_inst.v：AHT20 读取控制状态机。

十、经验总结

这个设计的关键在于把 IIC 基础时序和 AHT20 业务流程分层。底层 IIC 模块只负责起始、停止、读写和 ACK/NACK； 顶层 AHT20 状态机只负责命令流程、延时、CRC 和数据解析。这样后续更换其他 IIC 传感器时，可以复用底层 IIC 模块。

对 FPGA 来说，IIC 虽然是低速协议，但双向开漏 SDA、ACK 采样时刻、NACK + STOP 收尾、CRC 校验和传感器忙状态判断都容易出错。 因此调试时建议先验证单字节写、单字节读，再扩展到完整 7 字节读取流程。
