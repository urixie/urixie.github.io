RK3568 USB Ethernet Gadget 配置记录

本文记录在讯为 RK3568 开发板上配置 USB Ethernet Gadget 的过程。目标是在 Linux 系统中通过 USB 接口模拟出一个网络设备， 使开发板可以通过 usb0 与主机进行网络通信。

本文整理自我的原创调试记录，重点关注 RK3568 Linux 4.19 内核下 USB Gadget、RNDIS、ConfigFS、systemd 自启动和 usb0 静态网络配置。

一、实验环境

  • 硬件平台：讯为 RK3568 开发板
  • 内核版本：Linux 4.19.232
  • 文件系统：自制 Ubuntu Base 22
  • 功能目标：通过 USB Gadget / RNDIS 生成 usb0 虚拟网卡
  • 配置方式：内核 menuconfig + ConfigFS + systemd 服务 + usb0 静态 IP

二、功能目标

USB Ethernet Gadget 的作用是让开发板在 USB Device / OTG 模式下表现为一张 USB 虚拟网卡。 主机端识别后，可以通过该虚拟网卡与开发板通信，用于调试、数据传输、设备配置或无传统网口场景下的网络连接。

本次配置完成后，开发板侧会生成 usb0 网络接口，并配置为静态地址： 

usb0 IP：192.168.137.149/24
网关：192.168.137.1
DNS：8.8.8.8、114.114.114.114

三、内核配置路径

首先进入 Linux 4.19 内核配置界面，进入 USB Gadget Support 配置路径： 

Device Drivers
  -> USB support
    -> USB Gadget Support

关键配置包括 USB Gadget、ConfigFS、RNDIS 和 libcomposite。脚本中会检查以下配置： 

CONFIG_USB_CONFIGFS=y
CONFIG_USB_CONFIGFS_RNDIS=y
CONFIG_USB_LIBCOMPOSITE=y

四、USB Gadget 预组合配置

在 USB Gadget Support 中，还需要进入 USB gadget precomposed configurations，并选择 Ethernet Gadget。

五、系统侧配置思路

内核配置完成并烧录后，系统侧主要通过 ConfigFS 创建 USB Gadget 设备，并绑定 UDC 控制器。整体流程如下：

  1. 检查是否具备 root 权限。
  2. 检查内核是否启用了必要的 USB Gadget 配置。
  3. 挂载 ConfigFS。
  4. 在 /sys/kernel/config/usb_gadget 下创建 gadget 实例。
  5. 配置 idVendor、idProduct、manufacturer、product 等 USB 描述符。
  6. 创建 RNDIS 功能节点。
  7. 将 RNDIS 功能链接到配置项。
  8. 绑定 UDC 控制器，使 USB Gadget 生效。
  9. 等待 usb0 网络接口生成。
  10. 为 usb0 配置静态 IP、网关和 DNS。

六、自动配置脚本

下面的脚本用于在 Ubuntu Base 22 系统中自动创建 USB Gadget 初始化脚本、配置 systemd 服务，并生成 usb0 网络接口配置。 

#!/bin/bash
# Auto USB0 Network Configuration Script for Ubuntu Base 22 (Kernel 4.19)

# 检测root权限
if [ "$EUID" -ne 0 ]; then
    echo "错误：必须使用root权限运行此脚本！"
    exit 1
fi

# 定义配置参数
USB0_IP="192.168.137.149/24"
GATEWAY="192.168.137.1"
DNS_SERVERS=("8.8.8.8" "114.114.114.114")
UDC_DEVICE=$(ls /sys/class/udc 2>/dev/null | head -n1)

# 步骤1：验证内核配置
verify_kernel_config() {
    echo -e "\n[1/5] 验证内核配置..."
    if ! zcat /proc/config.gz | grep -Eq 'CONFIG_USB_CONFIGFS=y|CONFIG_USB_CONFIGFS_RNDIS=y|CONFIG_USB_LIBCOMPOSITE=y'; then
        echo "错误：缺少必要的USB Gadget内核配置！"
        echo "请确保以下配置已启用："
        echo "CONFIG_USB_CONFIGFS=y"
        echo "CONFIG_USB_CONFIGFS_RNDIS=y"
        echo "CONFIG_USB_LIBCOMPOSITE=y"
        exit 1
    fi
}

# 步骤3：创建初始化脚本
create_init_script() {
    echo -e "\n[2/5] 创建USB初始化脚本..."
    cat > /usr/local/sbin/usb-gadget-init.sh << EOF
#!/bin/bash

# 挂载ConfigFS
mount -t configfs none /sys/kernel/config 2>/dev/null

# 创建Gadget设备
mkdir -p /sys/kernel/config/usb_gadget/g1 || exit 1
cd /sys/kernel/config/usb_gadget/g1

# 设置设备描述符
echo 0x1d6b > idVendor
echo 0x0104 > idProduct
mkdir -p strings/0x409
echo "Linux Gadget" > strings/0x409/manufacturer
echo "RNDIS Ethernet" > strings/0x409/product

# 创建功能配置
mkdir -p functions/rndis.usb0 || exit 1
mkdir -p configs/c.1
ln -sf functions/rndis.usb0 configs/c.1

# 绑定UDC控制器
echo ${UDC_DEVICE} > UDC

# 等待接口生成
timeout=10
while [ ! -d /sys/class/net/usb0 ] && [ \$timeout -gt 0 ]; do
    sleep 1
    ((timeout--))
done

if [ ! -d /sys/class/net/usb0 ]; then
    echo "错误：usb0接口未生成！"
    exit 1
fi
EOF

    chmod 755 /usr/local/sbin/usb-gadget-init.sh
}

# 步骤4：配置Systemd服务
setup_systemd_service() {
    echo -e "\n[3/5] 配置系统服务..."
    cat > /etc/systemd/system/usb-gadget.service << EOF
[Unit]
Description=USB Gadget Configuration
After=sys-kernel-config.mount
Before=networking.service

[Service]
Type=oneshot
ExecStart=/usr/local/sbin/usb-gadget-init.sh
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target
EOF

    systemctl daemon-reload
    systemctl enable usb-gadget.service > /dev/null 2>&1
}

# 步骤5：配置网络接口
configure_network() {
    echo -e "\n[4/5] 配置网络接口..."
    cat > /etc/network/interfaces.d/usb0 << EOF
# USB0接口配置
auto usb0
iface usb0 inet static
    address ${USB0_IP}
    gateway ${GATEWAY}
    dns-nameservers ${DNS_SERVERS[@]}

    # 接口初始化优化
    pre-up ip link set dev usb0 up
    post-up route add default gw ${GATEWAY} dev usb0 metric 100
    pre-down ip route del default dev usb0
EOF
}

# 步骤7：启动服务并验证
final_check() {
    echo -e "\n[5/5] 启动服务并验证..."
    systemctl start usb-gadget.service
    systemctl restart networking.service

    echo -e "\n等待接口初始化(10秒)..."
    sleep 10

    echo -e "\n当前网络状态："
    ip -4 addr show usb0
    echo -e "\n路由表："
    ip route show
    echo -e "\nDNS配置："
    grep 'nameserver' /etc/resolv.conf
}

# 主执行流程
main() {
    verify_kernel_config
    create_init_script
    setup_systemd_service
    configure_network
    final_check

    echo -e "\n配置完成！请验证网络连接："
    echo "ping 192.168.137.1"
    echo "ping www.baidu.com"
}

# 执行主程序
main

七、网口冲突处理

配置 usb0 时需要注意：建议禁用其他网口，防止默认路由、DNS 或网关冲突，导致 usb0 虽然生成但无法正常通信。

如果开发板同时存在 eth0、wlan0、usb0 等多个网络接口，系统可能会自动选择其他接口作为默认路由。 这时即使 usb0 已经配置成功，也可能出现 ping 不通主机、DNS 不生效或访问外网失败的情况。

调试时可以临时关闭其他网口，只保留 usb0： 

ip link set eth0 down
ip link set wlan0 down

然后检查路由表： 

ip route show

确认默认路由是否走 usb0： 

default via 192.168.137.1 dev usb0 metric 100

八、验证方式

配置完成后，可以通过以下命令检查状态：

ip -4 addr show usb0
ip route show
cat /etc/resolv.conf
ping 192.168.137.1
ping www.baidu.com

如果 usb0 能正常获取静态地址，并能 ping 通主机侧网关，说明 USB 虚拟网卡基本配置成功。

九、经验总结

这次配置的关键不只是启用内核选项，还包括系统启动阶段的 ConfigFS 初始化、RNDIS 功能创建、 UDC 绑定、usb0 网络配置和路由冲突处理。

对于 RK3568 这类 Linux SOC 平台，USB Gadget 是一个很实用的调试和设备通信能力。它可以在没有以太网口、 Wi-Fi 未配置或需要稳定调试链路的场景中，为设备提供一个轻量的 USB 网络入口。
