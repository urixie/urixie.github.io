RK3568 SPI / spidev 功能调试记录

本文记录在讯为 RK3568 开发板上调通 SPI 功能的过程。实验环境为 Linux 5.10 内核和 Ubuntu Base 22 文件系统， 目标是在系统中生成 /dev/spidev0.0 设备节点，并通过用户态 C 程序完成 SPI 数据发送，最后使用逻辑分析仪验证波形。

本文整理自我的原创调试记录，重点关注 RK3568 SPI 引脚连接、内核 menuconfig 配置、设备树配置、spidev 用户态接口、/dev/spidev0.0 设备节点和逻辑分析仪验证。

一、实验环境

  • 硬件平台：讯为 RK3568 开发板
  • 内核版本：Linux 5.10
  • 文件系统：Ubuntu Base 22
  • 调试接口：SPI0
  • 用户态接口：spidev
  • 验证方式：C 测试程序 + 逻辑分析仪抓取 SPI 波形

二、硬件连接

SPI 转 CAN 模块需要接到 iTOP-RK3568 对应的 SPI 引脚。本文使用的 SPI0_M1 相关引脚包括：

  • SPI0_MISO_M1
  • SPI0_MOSI_M1
  • SPI0_CS0_M1
  • SPI0_CLK_M1

三、内核 SPI 配置

进入 Linux 5.10 内核配置界面，打开 SPI 支持相关选项： 

Device Drivers
  -> SPI support

需要重点确认以下配置：

  • 启用 Rockchip SPI controller driver。
  • 启用 User mode SPI device driver support。
  • 确保系统能够通过 spidev 向用户态暴露 SPI 设备节点。

四、设备树添加 SPI0 spidev 节点

内核配置完成后，还需要在设备树中开启 SPI0，并添加 spidev 设备节点。打开以下文件： 

rk3568_linux_5.10/kernel/arch/arm64/boot/dts/rockchip/topeet-rk3568-linux.dtsi

在文件最后添加如下内容： 

&spi0 {
        status = "okay";
        pinctrl-0 = <&spi0m1_cs0 &spi0m1_pins>;
        pinctrl-1 = <&spi0m1_cs0 &spi0m1_pins_hs>;

        mcp2515:mcp2515@0 {
                compatible = "rockchip,spidev";
                reg = <0>;
                spi-max-frequency = <10000000>;
                status = "okay";
        };
};

这里的 &spi0 表示启用 SPI0 控制器；pinctrl-0 和 pinctrl-1 选择 SPI0_M1 对应的普通速率和高速引脚复用配置。 子节点 mcp2515@0 挂在 SPI0 的片选 0 上，reg = <0> 对应 CS0，最终会在系统中生成类似 /dev/spidev0.0 的用户态设备节点。

这里虽然节点名写成 mcp2515，但 compatible 使用的是 rockchip,spidev，本质上是为了通过 spidev 暴露用户态 SPI 访问接口，方便先验证 SPI 总线通信。

五、确认 spidev 设备节点

内核、设备树配置完成并重新编译烧录后，启动系统并检查 /dev 下是否生成 spidev 设备节点： 

ls /dev/spidev0.0

如果可以看到 /dev/spidev0.0，说明系统已经暴露出 SPI 用户态设备。

/dev/spidev0.0 表示 SPI 总线上的一个具体设备。其中第一个数字 0 表示 SPI 总线编号， 第二个数字 0 表示连接在该 SPI 总线上的设备编号，通常与片选信号 Chip Select 对应。

六、用户态 SPI 测试程序

下面的程序通过 /dev/spidev0.0 打开 SPI 设备，设置 SPI 模式、每字比特数和最大速度， 然后通过 SPI_IOC_MESSAGE 向 SPI 总线发送 10 字节测试数据。 

#include <stdio.h>
#include <sys/ioctl.h>
#include <linux/spi/spidev.h>
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <string.h>

#define RESET   0xc0    // 复位命令
#define CANSTAT 0x0e    // CAN状态寄存器地址
#define READ    0x03    // 读命令
#define CANCTRL 0x0f    // CAN控制寄存器地址
#define WRITE   0x02    // 写命令

int fd;                     // SPI设备文件描述符
int mode = SPI_MODE_0;      // SPI模式
int bits = 8;               // 每字比特数
int speed = 10000000;       // 最大SPI总线速度，单位 Hz
int delay;                  // 延迟时间，单位 us

/*
 * 初始化SPI通信。
 * 返回0表示成功，-1表示失败。
 */
int spi_init(void)
{
    int ret;

    // 打开SPI设备文件
    fd = open("/dev/spidev0.0", O_RDWR);
    if (fd < 0) {
        printf("打开 /dev/spidev0.0 错误\n");
        return -1;
    }

    /*
     * 设置SPI模式
     */
    ret = ioctl(fd, SPI_IOC_WR_MODE32, &mode);
    if (ret == -1)
        printf("无法设置SPI模式\n");

    ret = ioctl(fd, SPI_IOC_RD_MODE32, &mode);
    if (ret == -1)
        printf("无法获取SPI模式\n");

    /*
     * 设置每字比特数
     */
    ret = ioctl(fd, SPI_IOC_WR_BITS_PER_WORD, &bits);
    if (ret == -1)
        printf("无法设置每字比特数\n");

    ret = ioctl(fd, SPI_IOC_RD_BITS_PER_WORD, &bits);
    if (ret == -1)
        printf("无法获取每字比特数\n");

    /*
     * 设置最大传输速度
     */
    ret = ioctl(fd, SPI_IOC_WR_MAX_SPEED_HZ, &speed);
    if (ret == -1)
        printf("无法设置最大传输速度\n");

    ret = ioctl(fd, SPI_IOC_RD_MAX_SPEED_HZ, &speed);
    if (ret == -1)
        printf("无法获取最大传输速度\n");

    printf("SPI模式: 0x%x\n", mode);
    printf("每字比特数: %d\n", bits);
    printf("最大速度: %d Hz (%d KHz)\n", speed, speed / 1000);

    return 0;
}

/*
 * 执行SPI数据传输。
 * 参数：
 *   fd  - SPI设备文件描述符
 *   tx  - 发送缓冲区
 *   rx  - 接收缓冲区
 *   len - 数据长度
 * 返回0表示成功，-1表示失败。
 */
int transfer(int fd, char *tx, char *rx, int len)
{
    int ret;

    struct spi_ioc_transfer tr = {
        .tx_buf = (unsigned long)tx,
        .rx_buf = (unsigned long)rx,
        .len = len,
        .delay_usecs = delay,
        .speed_hz = speed,
        .bits_per_word = bits,
    };

    ret = ioctl(fd, SPI_IOC_MESSAGE(1), &tr);
    if (ret < 1) {
        printf("无法发送SPI消息\n");
        return -1;
    }

    return 0;
}

int main(int argc, char *argv[])
{
    char reset_cmd[10] = {
        0x01, 0x02, 0x03, 0x04, 0x05,
        0x06, 0x07, 0x08, 0x09, 0x0a
    };

    spi_init();

    transfer(fd, reset_cmd, NULL, sizeof(reset_cmd));

    return 0;
}

七、编译与运行

将测试程序保存为 app.c，然后在开发板系统中编译运行： 

gcc app.c -o app
./app

如果 SPI 配置正常，程序会打开 /dev/spidev0.0，并通过 MOSI 发送测试数据。

八、逻辑分析仪验证

为了确认 SPI 总线确实有数据输出，可以使用逻辑分析仪抓取 CLK、MISO、MOSI、CS 信号。 本次测试中，MOSI 上可以看到发送的数据 01、02、03、04、05、06、07、08、09、0A，说明用户态程序已经成功驱动 SPI 发送数据。

九、经验总结

RK3568 上调通 SPI 功能时，需要同时关注硬件引脚、内核配置、设备树、spidev 设备节点和用户态测试程序。 其中 /dev/spidev0.0 是否生成，是判断用户态 SPI 通路是否打通的重要标志。

逻辑分析仪验证也很关键。仅仅程序运行不报错，并不代表 SPI 总线上一定有正确波形。 通过抓取 CLK、CS、MOSI、MISO，可以确认 SPI 模式、片选时序和数据内容是否符合预期。
