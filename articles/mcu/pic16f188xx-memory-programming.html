PIC16(L)F188XX 存储器编程规范整理

本文整理 PIC16(L)F188XX 系列微控制器的存储器编程规范，重点关注基于 SPI 兼容接口的 ICSP 编程方法、低压与高压进入模式、程序闪存和数据 EEPROM 的擦除写入流程，以及配置字、代码保护和 HEX 文件使用规则。

PIC16(L)F188XX 使用兼容 SPI 的新型编程方法，命令长度为 8 位。低压进入代码采用 32 个时钟，最高有效位优先，与部分旧系列器件存在差异。

一、编程接口与可编程区域

PIC16(L)F188XX 的非易失性存储器可以通过高压 ICSP 和低压 ICSP 两种方式进行编程。编程数据最终可写入程序闪存、数据闪存 EEPROM、用户 ID 区以及配置字区域。

  • 程序闪存 PFM：用于存放用户程序。
  • 数据闪存 EEPROM：用于存放掉电保持的数据。
  • 用户 ID 存储区：地址范围为 8000h～8003h，每个位置为 14 位。
  • 设备 ID / 版本 ID：设备 ID 位于 8006h，版本 ID 位于 8005h，均为只读。
  • 配置字：从 8007h 开始，决定系统振荡器、复位、看门狗、低压编程和代码保护等关键行为。

二、高压 ICSP 与低压 ICSP 的硬件差异

高压 ICSP 模式需要两个可控电源：一个用于 VDD，另一个用于 MCLR/VPP。进入编程模式时，需要在 MCLR/VPP 引脚施加高压 VIHH。低压 ICSP 模式只需要单一 VDD 电源，MCLR/VPP 引脚不需要特殊高压，只需按照低压进入时序保持对应电平。

低压 ICSP 是否可用由配置字中的 LVP 位控制。LVP 位出厂默认通常为 1，也就是允许低压编程。若将 LVP 位写为 0，则后续只能通过高压 ICSP 模式重新进入编程模式。

在低压 ICSP 模式下操作时，不能把 LVP 位写为 0。这个限制用于避免用户在低压编程过程中意外关闭低压编程入口，导致后续无法再通过低压方式访问器件。

编程相关引脚

  • VDD：器件电源。
  • VSS：地。
  • MCLR/VPP：复位 / 高压编程入口。
  • ICSPCLK：编程时钟。
  • ICSPDAT：双向编程数据线。

三、进入编程 / 校验模式

在编程 / 校验模式下，程序存储器和配置存储器可以通过串行方式访问。ICSPDAT 用于传输数据，ICSPCLK 用于提供时钟。命令和数据字均采用最高有效位优先传输，数据在 ICSPCLK 上升沿变化，并在下降沿锁存。

3.1 高压进入模式

高压进入方式有两种：VPP 优先进入模式和 VDD 优先进入模式。VPP 优先模式适用于需要防止器件先执行用户程序的场景，尤其是在 MCLR 被配置为普通 I/O、上电延时关闭、内部振荡器快速启动时更推荐使用。

  • VPP 优先进入：先保持 ICSPCLK 和 ICSPDAT 为低，再升高 MCLR/VPP 到 VIHH，最后给 VDD 上电。
  • VDD 优先进入：先保持 ICSPCLK 和 ICSPDAT 为低，再给 VDD 上电，最后将 MCLR/VPP 升高到 VIHH。
  • 退出模式：将 MCLR 从高压或低电平状态释放到退出时序要求的状态。

3.2 低压 LVP 进入模式

低压编程模式通过 MCLR 低电平和 32 位密钥序列进入。进入时需要将 MCLR 拉低至 VIL，然后在 ICSPDAT 上输入 32 位密钥序列。该序列为 32'h4D434850，对应 ASCII 字符 “MCHP”。 

低压进入密钥：
32'h4D434850

ASCII：
M C H P

传输顺序：
最高有效位 MSb 优先

需要注意，模式检测硬件只验证前 31 位，但必须输入最后一个时钟才能触发检测。低压模式维持期间，MCLR 必须保持在 VIL。退出低压编程模式时，将 MCLR 从低电平拉高到 VIH 或更高即可。

四、ICSP 命令与有效载荷格式

器件进入 ICSP 编程 / 校验模式后，编程主机可以发送 8 位命令。部分命令带有 24 位有效载荷，例如加载 PC 地址、加载 NVM 数据、读取 NVM 数据等。命令和有效载荷的所有时钟均由外部编程器产生，器件不会主动驱动 ICSPCLK。

对于带有效载荷的命令，主机发送 8 位命令后，还需要发送 24 个时钟完成后续数据传输。实际有效数据位长度小于 24 位，其余位置由起始位、停止位和填充位组成，以便兼容常见的 8 位 SPI 主机实现。

  • 主机到器件的载荷：起始位、停止位和填充位应驱动为 0。
  • 器件到主机的载荷：起始位、停止位和填充位为无关位，主机应忽略。
  • 所有有效数据仍然采用最高有效位优先传输。
  • 主机和器件都在 ICSPCLK 下降沿锁存 ICSPDAT。

五、NVM 数据加载与读取命令

5.1 为 NVM 加载数据

“为 NVM 加载数据”命令用于向编程数据锁存器加载待写入内容。对于程序闪存、配置存储器和用户 ID 区，加载的是 14 位指令字；对于 EEPROM 数据存储区，加载的是 8 位数据字节。

数据加载后并不会立即写入存储器，而是先暂存在内部锁存器中。只有在后续执行“启动内部定时编程”或“启动外部定时编程”命令后，锁存数据才会真正写入目标 NVM 区域。

5.2 从 NVM 读取数据

“从 NVM 读取数据”命令用于从当前 PC 地址输出 NVM 数据。读取时，ICSPDAT 会在第一个 ICSPCLK 下降沿进入输出模式，并在第 24 个时钟下降沿后恢复为输入高阻态。

读取数据时，起始位和停止位宽度仅为半个位时间，编程器应忽略这些位，只采集有效数据字段。若程序存储器启用了代码保护 CP，从程序存储区读取的数据会返回全 0。

六、地址控制：递增地址与加载 PC

PIC16(L)F188XX 的编程访问由程序计数器 PC 指向目标存储区域。地址可以通过“递增地址”命令自动加 1，但不能递减。如果需要跳转到指定地址，必须使用“加载 PC 地址”命令重新设置 PC。

  • 递增地址：PC 自动加 1，适合连续读取或连续加载数据。
  • 加载 PC 地址：直接设置 PC，用于切换到程序闪存、EEPROM、用户 ID 或配置字地址。
  • 写入行数据时，最终写入目标由启动编程时的 PC 所在物理行决定。

七、启动编程：内部定时与外部定时

7.1 启动内部定时编程

启动内部定时编程前，必须先通过“为 NVM 加载数据”命令加载待写入数据。命令发出后，器件内部定时电路会自动完成写入操作。主机必须等待规定的编程周期时间，确保写入完成后再发送下一条命令。

7.2 启动外部定时编程

外部定时编程同样需要先加载数据，然后在指定时间窗口内发送“结束外部定时编程”命令。需要特别注意，外部定时写入不支持配置位，对配置字执行外部定时写入不会改变目标字内容。

八、擦除策略：批量擦除与行擦除

8.1 批量擦除

批量擦除命令会根据当前 PC 地址决定擦除区域。收到批量擦除命令后，主机必须等待批量擦除时间结束。在擦除时间完全结束前，编程主机不得发送新的 8 位命令。

批量擦除通常有更高的最低 VDD 要求，高于普通行擦除和写入操作。做低压编程时尤其需要确认 VDD 高于器件要求，否则可能因为 BOR 或电压不足导致擦除失败。

8.2 行擦除

行擦除命令用于擦除当前 PC 指向的单个闪存行。PIC16(L)F188XX 的程序闪存和 EEPROM 均按行进行编程，行大小为 32 个 14 位字。行擦除结束前，主机必须等待规定的行擦除时间。

如果程序存储器启用了代码保护，行擦除程序存储器命令会被忽略。不过，当地址位于 8000h～800Bh 时，无论 CP 如何设置，行擦除命令只擦除用户 ID 存储区。

九、推荐编程算法

器件通过内部锁存器临时保存待写入的 14 位数据字。编程器通常先加载一行数据锁存器，再执行启动编程命令。写入操作不能跨越物理行边界，启动编程命令发出时的 PC 地址决定实际写入的目标行。

如果写入锁存器数量超过最大值，且未执行启动编程命令，锁存器中的旧数据可能会被覆盖。因此，编程器应严格按照行大小组织写入流程，完成一行加载后立即启动编程，并在编程结束后校验。

  • 程序闪存 PFM：按 32 字一行进行擦除和写入。
  • EEPROM 数据区：按行编程。
  • 用户 ID 和配置字：按单字编程。
  • 配置字建议使用内部定时写入，不使用外部定时写入。

十、代码保护规则

代码保护由 CP 位控制。使能代码保护后，外部编程器读取程序存储器 0000h～7FFFh 时会得到全 0，并且该区域的外部编程会被禁用，直到下一次批量擦除。器件运行时，用户程序仍然可以按器件规则访问程序存储器。

数据存储器保护由 CPD 位控制。CPD 置 0 时，EEPROM 数据存储器代码保护使能；CPD 置 1 时，保护禁用。无论 CP 和 CPD 状态如何，用户 ID 区和配置字均可编程和读取。

  • CP = 0：程序闪存代码保护使能。
  • CP = 1：程序闪存代码保护禁用。
  • CPD = 0：数据 EEPROM 代码保护使能。
  • CPD = 1：数据 EEPROM 代码保护禁用。
代码保护的解除方式不是直接改 CP 或 CPD，而是执行覆盖对应区域的批量擦除。设计编程器时，应把解除保护和整片擦除流程绑定处理。

十一、HEX 文件与校验和处理

Intel INHX32 格式的 HEX 文件中，每个程序字以 2 字节保存，并且低字节在前，高字节在后。由于每个程序字占两个字节，HEX 文件中的地址是器件程序存储器物理地址的 2 倍。 

示例：
配置字 1 物理地址：8007h
HEX 文件地址范围：1000Eh～1000Fh

编程器应支持从 HEX 文件中读取配置字和用户 ID。如果 HEX 文件缺少配置字信息，可以给出警告。若 HEX 文件中包含设备 ID，编程器应将其与实际读取到的设备 ID 比对，不匹配时应提示用户。

校验和计算思路

  • 代码保护禁用时：读取程序存储器内容并累加，再加入配置字有效位。
  • 代码保护使能时：可使用用户 ID 低半字节保存未保护状态下的 16 位校验和。
  • 未实现的配置位参与校验时应按掩码处理。

十二、电气规格与时序注意事项

编程 / 校验模式的交直流特性需要结合器件数据手册和编程规范共同确认。尤其是批量擦除、低压编程和 BOR 阈值相关条件，需要在编程器硬件设计阶段预留足够裕量。

  • 批量擦除后的器件默认使能 BOR，且 BORV 为低触发点。
  • 低压编程时，VDD 必须高于对应 BOR 阈值，避免器件持续复位。
  • 批量擦除要求 VDD 高于约 2.4V 的 BOR 门限，该门限与 BORV 配置位设置无关。
  • 配置位不支持外部定时写入。

十三、配置字重点整理

配置字决定器件复位后的基础运行状态。对编程器而言，配置字不仅影响用户程序运行，也会影响后续是否还能低压进入编程模式，因此必须谨慎处理。

13.1 振荡器与时钟相关配置

FCMEN、CSWEN、CLKOUTEN、RSTOSC 和 FEXTOSC 等配置位用于设置故障保护时钟监视、时钟切换、CLKOUT 输出、默认振荡器源以及外部振荡器模式。

13.2 调试、BOR 与 MCLR 配置

DEBUG、STVREN、PPS1WAY、ZCDDIS、BORV、BOREN、LPBOREN、PWRT 和 MCLRE 等配置位会影响调试功能、栈错误复位、PPS 锁定、过零检测、欠压复位、上电定时器和 MCLR 引脚功能。

13.3 看门狗配置

看门狗相关配置包括 WDT 时钟源、窗口选择和工作模式。WDTE 位用于决定 WDT 禁用、软件控制、仅运行态使能或始终使能等模式。

13.4 LVP 与写保护配置

LVP 位用于控制低压编程入口。LVP = 1 时，低压编程使能，MCLR/VPP 引脚作为 MCLR 使用，MCLRE 配置位会被忽略。LVP = 0 时，必须通过高压方式对 MCLR/VPP 施加 VIHH 才能进入编程模式。

WRT 位用于配置程序闪存自写和擦除保护，不同容量器件的保护范围需要结合器件型号确认。

13.5 CP 与 CPD 代码保护配置

CPD 控制 EEPROM 数据存储器代码保护，CP 控制程序闪存代码保护。保留位必须写入 1，避免产生不可预期行为。

十四、实现编程器时的工程建议

如果要基于 ESP32、STM32、FPGA 或上位机工具实现 PIC16(L)F188XX 编程器，建议将底层 ICSP 时序、命令封装、NVM 行缓冲、擦除写入流程和校验逻辑分层设计。

  • 底层驱动层：负责 ICSPCLK、ICSPDAT、MCLR/VPP 和 VDD 控制。
  • 时序层：统一处理命令发送、24 位载荷发送、读取窗口和延时等待。
  • NVM 层：封装加载 PC、读取 NVM、加载 NVM、启动编程、行擦除和批量擦除。
  • HEX 层：解析 INHX32 文件，提取程序区、EEPROM、用户 ID 和配置字。
  • 校验层：完成读回校验、设备 ID 比对和校验和计算。
  • 保护层：在写 LVP、CP、CPD、WRT 等危险配置位前增加明确确认。
对量产工具而言，LVP、CP、CPD 和 WRT 配置位应被视为高风险配置。建议在 UI 中单独提示，并在执行前生成即将写入的配置字摘要，避免误锁芯片或误开代码保护。

十五、总结

PIC16(L)F188XX 的编程接口虽然采用 SPI 兼容的 8 位命令形式，但实际编程流程涉及高压 / 低压进入模式、24 位有效载荷格式、PC 地址选择、行锁存器、内部 / 外部定时编程、批量擦除、代码保护和配置字约束。实现编程器时，不能只关注 SPI 收发，还必须完整处理进入时序、电压条件、擦除等待、写入行边界和读回校验。

在实际工程中，建议优先实现低压 ICSP 读取设备 ID、加载 PC、读取 NVM 和行写入流程；待基础通信稳定后，再加入高压进入、批量擦除、代码保护解除、HEX 文件解析和配置字写入功能。

整理方向：PIC16(L)F188XX · ICSP 编程 · NVM 擦写 · 配置字 · 编程器实现。
