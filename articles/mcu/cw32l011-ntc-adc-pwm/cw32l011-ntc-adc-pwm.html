CW32L011 NTC 热敏电阻采集与 PWM 频率输出

本文记录基于 CW32L011K8T6 的 ADC 采集 NTC 热敏电阻，并将采集结果转换为 PWM 输出频率的完整工程配置与代码分析。 实验在 CW32L011K8T6 StartKit 上完成，ADC 采集 12 通道热敏电阻分压值，取最小值计算 NTC 阻值，再映射为 PWM 频率由 PA12 输出。

一、功能概述

该工程实现了一个温度-频率转换链路：NTC 热敏电阻随温度改变阻值，通过分压电路将阻值变化转换为电压信号， CW32L011 的 ADC 模块采集该电压，计算出当前 NTC 阻值后查表映射为 PWM 频率，最后由 GTIM2 通道 3 输出对应频率的 PWM 波形。

数据流如下：

  1. ADC 顺序采样 12 个通道（AIN0～AIN11），每通道 10 次累加取平均
  2. 在 12 个通道结果中取最小值，认为是当前有效的 NTC 采样
  3. 将 ADC 值转换为电压，再通过 NTC 分压公式反推阻值
  4. 将阻值按经验公式映射为频率参数
  5. 用新频率参数重新配置 GTIM2 CH3，更新 PWM 输出频率
核心思路：NTC 阻值随温度降低而增大，分压点电压随之升高；ADC 测得电压后反向算出阻值，再查表得到目标 PWM 频率，最终在 PA12 引脚输出相应频率的 PWM。

二、硬件资源

  • 芯片：CW32L011K8T6（ARM Cortex-M0+，48MHz）
  • 开发板：CW32L011K8T6 StartKit
  • 时钟：HSI 48MHz，HCLK = PCLK = SysClk = 48MHz
  • ADC 时钟：PCLK 8 分频 = 6MHz，采样周期 262 ADC 时钟（≥40μs）

引脚分配

  • PA00～PA09、PB00～PB01：ADC 模拟输入通道 AIN0～AIN11，连接 NTC 热敏电阻分压电路
  • PA12：GTIM2_CH3，PWM 输出引脚
PA00～PB01 共 12 个引脚需要配置为模拟输入模式（PAxx_ANALOG_ENABLE()），否则 ADC 无法正确采样。

六、NTC 阻值到 PWM 频率的转换

求 ADC 最小值

在多通道采样结果中取最小值，作为当前有效的 NTC 采样值（排除干扰和异常）： 

// 在通道数组中求最小值（伪代码）
void Find_Adc_Min(uint16_t* adcArray, uint8_t len, uint16_t* minVal)
{
    *minVal = adcArray[0];
    for (uint8_t i = 1; i < len; i++) {
        if (adcArray[i] < *minVal)
            *minVal = adcArray[i];
    }
}

七、主循环逻辑

int main(void)
{
    // ============ 初始化阶段 ============
    // 1. 系统时钟初始化（HSI 48MHz，HCLK=PCLK不分频）
    SystemClock_Config();

    // 2. GPIO 初始化（PA12 配置为 GTIM2_CH3 PWM 输出）
    GPIO_Config_PWM();

    // 3. 滴答定时器初始化（用于延时）
    InitTick(48000000);

    // 4. 使能 GTIM2 外设时钟
    __SYSCTRL_GTIM2_CLK_ENABLE();

    // 5. 初始化 PWM，默认输出频率（如 25820 Hz）
    PWM_Output_Config(DEFAULT_FREQUENCY);

    // 6. ADC 初始化（12 通道 GPIO 配置）
    ADC_GPIO_Config();

    // ============ 主循环 ============
    while (1) {
        // 1. 多通道 ADC 累加平均采样
        ADC_MultiChannel_Average();

        // 2. 取最小值作为有效 NTC 采样
        uint16_t adcMin;
        Find_Adc_Min(adc_avg, CHANNEL_MAX, &adcMin);

        // 3. NTC 阻值计算 → PWM 频率参数
        uint32_t freqParam;
        NTC_To_Frequency(adcMin, &freqParam);

        // 4. 用新频率参数更新 PWM 输出
        PWM_Output_Config(freqParam);

        // 5. 延时（采样间隔）
        SysTick_DelayMs(50);
    }
}

主循环每约 52ms 更新一次 PWM 频率（50ms 延时 + ADC 采样耗时）。 频率变化范围由 NTC 阻值范围和经验公式共同决定，对应典型的热敏电阻温度特性曲线。

八、总结

  • CW32L011 的 ADC 支持最多 16 通道序列采样，本例使用 12 通道分两批读取
  • ADC 采样周期设置为 262 个 ADC 时钟（≥40μs），满足 BGR 采样保持时间要求
  • GTIM2 CH3 通过 PWM1 模式输出占空比 50% 的 PWM 波形
  • 定时器时钟频率可通过实测补偿修正，使输出频率更准确
  • NTC 电压-阻值-频率映射公式取决于具体分压电路参数，需根据实际硬件调整
