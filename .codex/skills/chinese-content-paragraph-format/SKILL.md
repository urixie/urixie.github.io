# 中文内容段落排版规范

## 适用范围

本规范适用于网页中的“内容类文字”，包括：

- 首页介绍段落
- 技术栈说明段落
- 项目卡片说明段落
- 关于本站说明段落
- 右侧信息卡片说明段落
- 文章页正文段落
- 文章页提示块、警告块中的说明文字
- 图片说明文字
- 其他以阅读为主的中文说明性内容

## 不适用范围

以下内容不要做首行缩进：

- 页面标题，例如 h1、h2、h3、h4
- 导航菜单
- 按钮文字
- 标签、徽章、技术 tag
- 邮箱、链接按钮
- post-meta
- 英文副标题
- 平台分类标题
- 代码块 pre / code
- 表单控件
- 纯英文短标签
- 技术参数列表中非常短的 UI 项

## 核心排版规则

中文正文段落应使用 CSS 统一控制，不要在 HTML 文本前手动添加全角空格。

推荐规则：

```css
text-indent: 2em;
text-align: justify;
overflow-wrap: break-word;
```

其中：

- `text-indent: 2em;` 表示首行缩进两个中文字符。
- `text-align: justify;` 用于优化中文段落两端对齐效果。
- `overflow-wrap: break-word;` 用于避免长英文、路径、型号字符串撑破布局。

## 注意事项

1. 不要给标题、导航、按钮、标签、代码块添加首行缩进。
2. 不要在 HTML 正文开头手动加两个空格或两个全角空格。
3. 尽量通过统一 class 或 CSS 选择器完成排版。
4. 文章页正文的 `<p>` 应首行缩进。
5. 首页卡片中的说明性 `<p>` 应首行缩进。
6. `.article-note`、`.article-warning` 这类整段说明块，如果内容是中文说明，也应体现中文段落缩进。
7. `figcaption` 如果是完整中文说明句，也可以按内容段落处理，但要保持视觉效果，不要破坏图片说明的居中布局。
8. 代码块、命令行、路径、配置片段绝对不要被缩进影响。

## 验收标准

- 首页所有说明性段落首行缩进两个中文字符。
- 所有文章页正文段落首行缩进两个中文字符。
- 提示块、警告块中的中文说明符合中文段落阅读习惯。
- 导航、标题、按钮、标签、代码块没有被错误缩进。
- 移动端布局不被破坏。
